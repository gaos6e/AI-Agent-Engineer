# 误导风险与可访问性

## 本节目标

从坐标、分母、筛选、颜色、文字与导出文件六个层面审计图表，并建立“渲染—程序检查—人工读图—回改”的闭环。

## 常见误导

- 条形图截断 Y 轴，使微小差异看似巨大。
- 改变直方图 bin 只展示支持结论的形状。
- 只筛选成功运行后比较延迟。
- 用累计值掩盖近期退化，或用百分比隐藏绝对数量。
- 3D 透视让后排柱被遮挡、面积难比较。
- 双 Y 轴任意调整范围制造相关。
- 画平滑趋势但不展示原始波动与异常。

审查时问：数据如何筛选？分母是什么？缺失去哪了？坐标是否可比？聚合是否掩盖子群？结论换一种合理参数是否仍成立？

## 颜色与可访问性

颜色不应是唯一编码。安全/失败可同时用形状、线型、文字或直接标签。使用感知均匀、色盲友好的 colormap；有序数据用连续色带，类别用离散颜色，正负偏差用有明确中心的 diverging 色带。

W3C WCAG 2.2 的“Use of Color”要求不能只靠颜色传达信息。数据图可用“颜色 + marker/线型/纹理/直接标签”冗余编码。单向连续值可从 `viridis`、`magma`、`cividis` 等感知较均匀色图起步；有正负且 0 是业务中心时才用发散色图并锁定对称范围。不要用 `jet/rainbow` 制造不均匀亮度边界。

确保文字与背景对比充分，字号可读，图例靠近数据。对外发布时提供简短替代文本：图展示什么、主要趋势、关键异常和数据范围。

## 标注与来源

每张可交付图至少有标题、轴标签、单位、图例/直接标签、时间窗口、样本数或分母、误差条含义、数据/代码版本。标题可陈述观察，不应夸大因果。

替代文本不是逐项朗读颜色和装饰，而是说明图型与数据范围、主要比较、关键异常、分母/区间和读图限制。复杂图还应提供底层表格或可下载数据。文字结论必须由同一份冻结数据生成，避免图更新后替代文本仍保留旧数字。

## 可复现导出与视觉自检

1. 用最终交付尺寸创建 `Figure`，统一字体、单位、颜色语义和面板顺序。
2. 先渲染高分辨率 PNG 预览，同时保留 SVG/PDF 矢量版本；数据图不使用 JPEG。
3. 让程序检查缺字 warning、画布边界、刻度重叠、面板数量、像素尺寸和输出签名。
4. 实际打开 PNG，检查图例是否压数据、文字/色块对比、子图对齐、灰度下是否仍能辨认。
5. 修改后重新渲染；最终文件应能由锁定依赖、输入快照和脚本重建。

`bbox_inches="tight"` 或 constrained layout 是兜底，不是人工读图的替代品。特别是热力图格子文字可能在深浅背景上失去对比，程序没有理解语义时也可能“形式通过、读不清”。

## 发布前清单

- [ ] 条形长度从合理共同基线开始。
- [ ] 颜色在灰度或色觉差异下仍可辨认。
- [ ] 未隐藏失败、超时、缺失和小样本。
- [ ] 不确定性、筛选和聚合方法写在图注。
- [ ] 图中数字可从底层数据重新计算。
- [ ] 导出尺寸下无裁切、重叠或乱码。

## 练习

找一张内部评测图，分别从坐标、分母、缺失、颜色、来源五方面审计。把结论改写成只包含图中证据能支持的范围。

## 掌握检查

- [ ] 我能指出截断坐标、选择性分箱、成功样本筛选和双 Y 轴的误导路径。
- [ ] 我不会只靠颜色区分类别，会检查灰度与直接标签。
- [ ] 我能为连续、发散和分类数据选择合适色图并给 colorbar/单位。
- [ ] 我会同时导出可预览位图与矢量图，并在最终尺寸实际读图。
- [ ] 我能写包含主要趋势、关键异常、分母与限制的替代文本。
