# 项目：最小向量检索器

## 项目目标

运行 [[向量基础/examples/vector_search.py|vector_search.py]]，只用 Python 标准库实现 dot、L2 norm、单位化、cosine、Euclidean distance、exact top-k 和 Recall@k。[[向量基础/examples/test_vector_search.py|test_vector_search.py]]用 9 项测试固定数学关系、排序方向、异常输入和命令行行为。向量是人为构造的三维教学数据，不代表真实 embedding 模型。

## 环境与运行

项目没有第三方依赖。为了形成稳定习惯，仍建议在 Windows 11 / PowerShell 7 中使用独立 `venv`，且把环境放在 vault 外：

```powershell
$exampleDir = (Resolve-Path '.\Knowledge\AI Agent Engineer\docs\向量基础\examples').Path
$venv = Join-Path $env:LOCALAPPDATA 'Gao-venvs\vector-search'
$python = Join-Path $venv 'Scripts\python.exe'

py -3.11 -m venv $venv
& $python -m pip --version
& $python -B -W error (Join-Path $exampleDir 'vector_search.py')
& $python -B -O -W error (Join-Path $exampleDir 'vector_search.py')
& $python -B -W error -m unittest discover -s $exampleDir -p 'test_*.py' -v
& $python -B -O -W error -m unittest discover -s $exampleDir -p 'test_*.py' -v
```

这里不需要安装包；`pip --version` 只是确认后续依赖会进入正确环境。`-B` 禁止生成字节码缓存，`-O` 会移除普通 `assert`。脚本把关键教学检查写成显式异常，因此普通和优化模式都保留相同验收行为。

## 数据、输出与解释

query 是 `(1, 1, 0)`，三个文档向量分别代表 Python API、HTTP 重试和猫咪护理。脚本在 cosine 与 Euclidean 两种度量下都把前两个 API 文档排进 top-2：

| 度量 | 第 1 名 | 第 2 名 | 排序方向 |
| --- | --- | --- | --- |
| cosine | `python-api`，约 0.9986 | `http-retry`，约 0.9948 | 越大越相似 |
| Euclidean | `python-api`，约 0.1000 | `http-retry`，约 0.2449 | 越小越相似 |

人工相关集合是 `{python-api, http-retry}`，所以 `Recall@2=2/2=1.0`。这只证明三条人工数据上的实现与标注一致；它不证明真实语义模型、ANN 索引或 RAG 回答质量。

## 常见错误与排查

| 错误 | 后果 | 排查与修正 |
| --- | --- | --- |
| cosine 按升序排 | 最不相似结果被排在前面 | 先写清“相似度越大越好、距离越小越好” |
| 只单位化 query | inner product 不再等价于 cosine | query 与文档执行同一归一化约定 |
| 把零向量替换为很小常数后继续 | 隐藏上游数据或模型故障 | 明确拒绝并记录来源，修复生成链路 |
| 使用 `==` 比较浮点结果 | 舍入误差导致脆弱测试 | 用绝对/相对容差，并测试排序与边界 |
| 从检索分数生成“相关标签” | 评测只会重复模型偏好 | 由任务定义和人工/权威数据建立 ground truth |
| exact 小数据结果直接外推 ANN | 忽略索引漏召回与过滤成本 | 以 exact top-k 为基准测 ANN Recall@k、延迟和内存 |
