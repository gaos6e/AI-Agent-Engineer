# 成本、失败恢复与来源治理

## 本节目标

控制长任务的重试成本，保留逐镜头来源链，并在发布前完成安全、权利与可恢复性检查。

## 预算按镜头管理

视频成本受模型、时长、分辨率、候选数、扩展/编辑轮次、失败请求、存储、下载、转码和人工时间影响。价格属于动态事实，运行前读取供应商官方页面或账单，不在知识库写死数字。每个镜头设置 `max_attempts`，项目设置总候选时长、总尝试和人工审批阈值。

草稿阶段使用满足构图验证的最低成本规格；只有镜头内容通过后才做高规格或后期。报告“每个批准秒需要多少生成秒/尝试”，能揭示只看单次报价看不到的浪费。

## 检查点与局部恢复

检查点可以是：分镜批准、各镜头批准、画面锁定、声音锁定、成片批准。作业失败后从最近检查点继续。连续三次相同视觉失败时暂停并转人工拆镜头；安全拒绝、输入权利缺失、人物同意不明立即停止，不通过改写描述绕过。

对异步 API 保存内部请求 ID 与供应商 job ID；连接中断先查状态。下载使用临时文件和哈希；后期链每一步记录父资产与 `transform_id`。删除供应商或本地临时素材前，确认保存期、审计义务和用户撤回要求。发生撤权/删除时，不只删原文件：按 `asset_id → transform_id → release_id` 使镜头候选、代理文件、成片派生物、缓存、搜索索引和公开链接传播失效，并记录最小化处理结果。

## 来源与内容凭证

逐镜头 manifest 记录输入 `asset_id`、`source_revision`、权利和对象级授权/ACL 决定、请求与模型元数据、生成时间、选择理由、编辑动作与 `transform_id`、音频/字幕来源、输出哈希、批准人及 `release_id`。C2PA 2.4 可为媒体 provenance 声明提供签名结构，但不会证明画面里的事件发生过，也不能替代对象级授权/ACL、人物同意、版权核查或人工审查。拼接、转码和平台上传可能剥离或改变凭证，发布后需再次验证。

内容凭证、哈希和供应商元数据说明可验证的来源声明，不是发布许可，更不是事实证明。若成片用于对外新闻、医疗、科学或身份主张，应把该主张关联到独立来源和 `release_id`；仅当证据可支持时才标为 `evidence_supported`。

将旁白脚本、分镜说明或评审任务设为 `evidence_bound`，表示它们只能引用已批准的证据集合；这能约束输入，却不自动使生成视频或发布结论成为 `evidence_supported`。后者仍需要针对具体外部主张的独立证据和人工发布判断。

## 发布前安全检查

- 真实人物、声音或可识别身份是否有明确同意？
- 是否含未授权角色、音乐、商标、素材或机密信息？
- 是否可能被误当作真实事件、新闻、证据或专业建议？
- 事实性陈述是否有独立证据达到 `evidence_supported`，而非仅由生成画面、内容凭证或相似度分数支撑？
- 是否有暴力、性、仇恨、未成年人或高风险场景？
- 是否完成披露、字幕、无障碍、来源、对象级 ACL 与撤权/删除传播渠道？

供应商政策是最低边界之一，不代表应用场景自动安全。NIST AI 600-1 提供跨生命周期风险管理框架，可帮助把风险、测量、责任人与处置证据写入项目，而不是只依赖一次过滤结果。
