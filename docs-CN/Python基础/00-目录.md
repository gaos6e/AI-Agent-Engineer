# Python基础

## 知识库简介

本知识库由两层组成：原有 **Python 100 天**公开课程作为冻结参考层；新增 `Agent工程路线/` 作为本路线独立维护的必修层。目标不是把 100 天内容逐页背完，而是先掌握“环境—数据边界—可靠性—测试—并发—工具契约”这条工程主线，再按项目需要回到公开课程学习 Web、数据库和数据分析。

> [!warning] 原教程图片路径
> 原教程有少量图片链接写死了重命名前的 `Python-100-Days-master` 目录名。为同时遵守“原教程正文不改”和“不要恢复旧目录”的要求，本轮未改写这些原文件，也未创建旧目录别名；若个别旧页面图片未显示，请在当前课程同级的 `res/` 目录中查看原资源。新增路线中的 wikilink 已按当前 `Python基础` 路径验证。

> [!info] 资料时点与兼容性
> Python、第三方库和安装方式会变化。本入口于 **2026-07-14** 核对：Python.org 的最新稳定版为 **Python 3.14.6**；Python 3.15 仍为预发布分支，不作为默认生产基线。原教程包含不同时期的生态示例，运行前应先读对应依赖说明，并以项目锁定版本和当前官方文档为准。新增路线尽量兼容 Python 3.11+；配套示例的实际本机验证版本与结果见本页“验证记录”，不能据此声称所有 Python 版本均已验证。

> [!danger] 冻结参考层中的已知不可运行或不安全示例
> 以下问题于 2026-07-19 在本地审计确认。它们保留用于理解历史 API，不是当前运行指引：
>
> - [[Python基础/Day31-35/31.Python语言进阶|Python 语言进阶]]把裸 coroutine 交给 `asyncio.wait()`；在本项目 Python 3.11.9 会直接抛出 `TypeError: Passing coroutines is forbidden`。现代代码应先创建 Task，或根据取消与错误语义选择 `gather()` / `TaskGroup`，不要原样运行该片段。
> - `Day31-35/code/example24.py` 对真实站点使用 `ssl=False`，又缺少连接/总超时、状态码与响应大小检查；不要关闭证书校验或直接请求示例中的站点。练习时改用本地 fixture/测试服务器，并显式验证超时、状态、大小和解析失败。
> - `公开课/年薪50W+的Python程序员如何写代码/code/Python/opencourse/part01/example10.py` 会连续请求真实第三方页面，硬编码 `BaiduSpider` User-Agent，且没有 timeout、状态检查或站点政策核对。不要直接运行；教学测试默认使用本地 HTML fixture，真实采集必须使用诚实可联系的身份、限速，并先核对 robots 与服务条款。
> - 公开课的 `requirements.txt` 是 2020 年上游环境快照，含多组旧版和平台相关依赖。不要在现代环境直接执行 `pip install -r`；只在隔离环境为当前练习声明最小依赖、重新解析并测试。
> - `Day31-35/code/example01.py` 的查找算法现在可在无绘图库环境中导入和测试；只有执行 `main()` 绘制复杂度曲线时才需要 `matplotlib` 与 `numpy`（可在隔离环境运行 `python -m pip install matplotlib`）。
>

## 在总路线中的位置

Python 是工程基础中的编程主线。后续 API、数据清洗、RAG、Tool Calling、MCP 和 Agent 框架都需要你能看懂函数、字典/列表、异常、文件、JSON、模块和测试。先完成本页的“最小必修”，再进入 API 与 LLM 应用；Web 框架和数据科学章节可以按需回补。

## 学习目标

- 在 Windows 11 + PowerShell 7 中创建 `venv`，用 `python -m pip` 安装依赖。
- 使用变量、分支、循环、函数、模块、列表、字典和集合表达任务逻辑。
- 用类型提示和数据类表达内部模型，同时对不可信输入做运行时校验。
- 读写显式 UTF-8 与 JSON，限制资源并区分语法、结构和业务错误。
- 设计异常、总时限、有限重试、日志、测试、取消和并发上限。
- 构建可复现运行的 CLI，并为 API、RAG 或 Agent 工具写“小而深”的 Python 执行边界。

## 前置知识

- 会在资源管理器中定位文件夹，并能在 PowerShell 中运行命令。
- 不要求编程经验。遇到旧版截图或库 API 时，先理解概念，再核对当前文档。

## 开始前：环境与版本

在独立练习目录中运行：

```powershell
python --version
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

如果 `python` 不是预期解释器，先用 `Get-Command python` 查看来源。`.venv` 是本机环境，不应加入 vault 或 Git。只有明确需要更快的环境/依赖管理时，再学习 `uv`；初学阶段先理解 `venv + pip` 的职责。
