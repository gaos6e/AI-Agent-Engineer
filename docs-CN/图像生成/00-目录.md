# 图像生成

## 知识库简介

图像生成（image generation）是让模型依据文本、已有图像或遮罩产生新像素的技术。本知识库不把“写一句提示词”当作完整工程：你会先理解任务和生成原理，再把提示、输入权利、质量标准、人工复核、成本与来源信息组织成可审计的工作流。每个输入和输出还要有可追溯的版本、对象级访问控制，以及可执行的撤权/删除传播。课程不要求本地显卡，也不会下载模型或真的生成图片。

## 在总路线中的位置

它位于“扩展应用与复杂协作”阶段。前面的语言模型与 Agent 能力负责理解需求、调用工具和管理状态；这里负责把视觉任务变成受约束、可验收的生成作业。

## 学习目标

- 区分文生图、图生图、局部编辑、扩图和变体等任务。
- 用直觉解释自回归与扩散生成，并知道“随机”为什么不等于“不可管理”。
- 把主体、构图、风格、文字、编辑约束和验收标准写成结构化任务。
- 设计供应商无关的 API 边界、质量评审、安全审核与复现记录；区分 `source_revision`、`transform_id` 与 `release_id`。
- 能运行离线审计器，在调用任何模型前发现缺失的权利、风险与验收信息。

## 掌握标准

- [ ] 能根据需求选择生成、编辑、局部重绘或扩图，而不是只说“做张图”。
- [ ] 能解释提示遵循、视觉质量、文字正确性和权利合规为何需要分别验收。
- [ ] 能写出含输入来源、风险、预算、复现和人工复核的任务清单。
- [ ] 能说明自动指标的适用边界，并设计盲评或成对比较。
- [ ] 能在不暴露密钥、不写死供应商参数的前提下设计生成接口。
- [ ] 能在候选评分前执行对象级授权/ACL，并在撤权或删除时让派生输出、缓存和链接按计划失效。
- [ ] 能说明内容凭证只提供可验证的来源声明；事实性发布必须做到 `evidence_supported`，不能由生成图本身单独支撑。
- [ ] 离线审计器及其测试可以通过。
