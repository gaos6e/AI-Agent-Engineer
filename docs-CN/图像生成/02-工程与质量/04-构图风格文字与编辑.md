# 构图、风格、文字与编辑

## 本节目标

把“视觉效果”拆成布局、视觉语言、文字层和编辑保护区，掌握模型不稳定时的工作流降级方案。

## 构图先于装饰

构图决定主体在画布中的位置与视觉层级。任务中至少写清：

- 主体数量、相对大小和位置；
- 视角、景别与地平线；
- 前景、中景、背景分别有什么；
- 标题、按钮或裁切所需的安全区；
- 视线、动作或光线引导的方向。

“三分法”“居中构图”是启发式，不是硬规则。真正的验收是下游用途是否成立：商品是否完整、操作步骤是否可读、缩略图里主体是否仍然突出。

## 用视觉特征描述风格

“像某位艺术家”既可能含糊，也可能带来权利和平台政策问题。优先描述可见特征：媒介（剪纸、铅笔、3D）、线条（粗轮廓、无轮廓）、色板（低饱和冷色）、材质（哑光纸张）、照明（柔和侧光）、细节密度和时代设计语言。保存合法的品牌色、字体许可证和示例资产来源。

## 文字生成要有后备路径

图内文字同时要求字符正确、排版合理和视觉融合。即使当前模型改进明显，也应逐字符核对，尤其是中文、数字、药名和网址。可靠工作流通常是：

1. 先生成留出文字安全区的无字底图；
2. 用排版工具添加正式文字；
3. 若必须让文字与场景融合，再把“带字版”作为候选，并保留无字版；
4. OCR 只能辅助检查，最终由人对照源文案。

## 编辑任务的保护契约

局部编辑至少包含 `asset_id`、`source_revision`、`source_hash`、`mask_hash`、`editable_region`、`protected_region`、`desired_change` 和差异验收。实际执行和后期分别写入 `transform_id`，待发布候选另有 `release_id`；这三类 ID 不能由文件名或提示文本反推。不要只看目标区域是否变好，还要做**保护区回归**：人脸、Logo、产品标签或科学图中不应改变的像素是否漂移。

输入图像经过缩放、自动旋转或色彩空间转换后，遮罩可能错位。排查顺序是：核对宽高与方向元数据，再可视化遮罩边界，最后才调提示词。对参考图的读取、自动评分和人工预览都应先经过对象级授权/ACL；发生撤权/删除时，需沿着 `asset_id → transform_id → release_id` 传播到候选、缓存和发布链接。

## 常见错误与排查

- 主体被裁：在任务层记录安全区，并用实际投放模板预览。
- 风格压过内容：先锁构图与信息，再加风格。
- 文字近似正确：逐字符比对，不以“看起来像”为准。
- 编辑污染全图：降低修改范围，保存基线图并做差异检查。
- 颜色不一致：记录色彩空间和后期导出设置，品牌色由后期校准。

## 练习与自测

为“竖版课程封面”画一个文字布局框（纸笔即可），写出安全区、主体框和裁切规则。再把“赛博朋克风格”改写为至少五个可见特征，并说明哪些属于软偏好。
