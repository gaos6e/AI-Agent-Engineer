# 2026-07-18 内容审计与重构记录

## 结论摘要

本轮结论不是“原有 53 门课全部重写”。基线检查表明，现代主线课程的首页、来源边界、实践项目和掌握标准整体已经较成熟；主要问题是：总路线把全部课程呈现为单一线性序列，核心能力仍有三处明确缺口，图示与真实运行结果集中在参考层而未进入主线，网站又把 `53` 门课、`8` 个阶段和连续整数顺序写死。

因此采取以下策略：

1. 保留原 53 个课程入口，不做无证据删除或批量改写；
2. 新增“现代 LLM 能力与模型选择”“环境型 Agent”“实时多模态交互”3 门课程，补齐明确缺口；
3. 将总路线改为课程地图、角色路径和内容层级，不再要求人人线性完成全部课程；
4. 对 Agent、RAG、评测、安全、多 Agent、LLMOps 等高价值主线做精确补图，对数据可视化项目实际生成并嵌入结果；
5. 更新 MCP、Agent Skills、A2A/AG-UI 的稳定、扩展、实验和观察边界；
6. 将尚未完成来源核验的机器学习速查页明确降级，不借“全库优化”名义重写全部历史镜像；
7. 将网站课程发现、阶段统计和导航改为由元数据动态生成，为后续增删课消除硬编码阻碍。

> [!important] 决策边界
> “保留”不等于证明每一段历史内容都准确；“降级为参考”也不等于内容毫无价值。本轮只对有足够证据且影响主线的内容直接修改。未经逐页核验的镜像、法律结论、第三方图片许可和快速变化协议不会被写成稳定事实。

## 证据层级

本轮按以下优先级作判断：

1. **当前仓库事实**：文件、frontmatter、索引、脚本、fixtures、测试、生成清单和发布规则；
2. **标准与第一方资料**：MCP、Agent Skills、JSON Schema、W3C、IETF、NIST、OWASP、MITRE、Playwright 等官方规范或文档；
3. **原始论文与官方项目**：HELM、Model Cards、WebArena、OSWorld、SWE-bench、RAG、MTEB 等；
4. **供应商动态文档**：只用于核对当日 API、模型目录、版本与实现样例，不上升为供应商无关结论；
5. **既有整理与工程推断**：作为定位线索；无法被更高层证据确认时，标为观察、实验、待核验或未知。

对易变化内容采用四类状态：

| 状态 | 含义 | 处理方式 |
| --- | --- | --- |
| stable | 已发布规范、长期工程不变量 | 可进入核心掌握标准，但仍记录版本与日期 |
| dynamic | 模型目录、SDK、价格、配额、区域和产品 API | 只保留核验入口与选择方法，接入当日重查 |
| frontier/observed | 草案、实验扩展、新协议或近期预印本 | 独立专题或观察表，不作为稳定前置 |
| needs-review | 历史页尚未逐项核对来源、公式或实现 | 从主线降级并加警告，不继续传播为默认做法 |

## 抽样与限制

- 全量完成了目录、课程首页、元数据、文件类型、示例/测试命名、Mermaid/图片分布和公开规则检查；没有逐句事实核验 844 篇 Markdown。
- 深读优先覆盖 Agent 核心、Tool Calling、MCP、Agent Skills、RAG、评测、LLMOps、运行监控、安全、治理、多 Agent、数据可视化和机器学习入口；参考镜像按风险抽样。
- “有参考资料标题”只是结构证据。部分课程使用“下一步与参考”等不同标题，不能因关键词统计缺失就判定无来源。
- 公开 benchmark、论文实验和供应商说明只能帮助发现候选与失败模式，不能证明本地业务、权限、语言、成本和延迟条件。
- 测试文件数不能证明覆盖率；离线模拟器也不能证明真实浏览器、桌面、音频栈、云服务或模型行为。
- 法规、跨境数据、录音同意、许可证和第三方再分发需要相应负责人判断，本轮不提供法律结论。
- Python 冻结参考层的相对链接抽查仍发现约 103 个历史断链和 3 个 stub 过渡问题；它们不阻断原创主线，但若继续公开该层，需要单独修复或归档决策。

## 为什么本轮没有删除或物理合并课程

删除判断为 **0 门**，物理合并判断为 **0 门**。理由不是保守地拒绝改变，而是当前证据支持“降低导航优先级”多于“内容没有保留价值”：

- 历史完整教程仍可用于补基础、研究或离线查阅；主线问题来自入口权重，不仅是文件存在。
- 框架/官方资料镜像易过时，但其中的术语、迁移线索和案例仍有参考价值；公开站点已经使用 stub/发布策略限制大体积层。
- 物理删除或改名会破坏 wikilink、相对链接、附件和外部书签；在未完成使用量、许可、替代页和重定向审计前，收益不足以覆盖风险。
- RAG 子课、数据课和多模态子课有不同输入契约、测试与失败责任层；合成一篇“大而全”页面反而降低教学和排障效果。
