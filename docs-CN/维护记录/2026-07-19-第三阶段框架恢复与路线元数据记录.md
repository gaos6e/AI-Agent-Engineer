# 2026-07-19 第三阶段框架恢复与路线元数据记录

本阶段把第二阶段队列中的两项高优先级工作变成可运行证据：为 LangGraph 与 CrewAI 分别增加真实 runtime 的 Layer B 恢复项目，并让第一批学习路线关系从手写说明进入严格校验的元数据。主 Agent 统一状态、幂等、恢复、来源和路线语义；专项 Agent 只读审计版本 API、课程衔接和验证边界，最终修改与冲突处理仍由主 Agent 完成。

> [!important] 结论边界
> 本阶段完成的是框架恢复实践和路线元数据的首批迁移，不表示全部 56 门课程、所有框架参考页或全部源文档都已逐页深审。真实模型调用、多 worker 并发、生产数据库、桌面端视觉检查及其余课程迁移仍在后续队列中。

## 1. LangGraph：从概念说明到真实可恢复审批流

- 新增 [[LangChain/00-初学者路线/08-项目-LangGraph可恢复审批流|项目：LangGraph 可恢复审批流]]，原测试与升级清单后移为第 09 节，并同步目录与前后导航。
- 新增 `examples/langgraph_layer_b/`：直接运行 `langgraph==1.2.9`、`langgraph-checkpoint-sqlite==3.1.0`，隔离环境解析到 `langgraph-checkpoint==4.1.1` 与 `langchain-core==1.4.9`。
- 示例用真实 `StateGraph`、`SqliteSaver`、`interrupt()`、`Command(resume=...)` 和 `durability="sync"`，验证跨数据库连接和跨 Python 进程恢复。
- 应用包装层在 resume 前检查 thread 是否存在、受信任 `owner_id` 绑定、目标节点与 pending interrupt，避免把 runtime 对未知或已完成 thread 的默认行为误当成授权拒绝；示例 owner 参数只模拟认证边界，不替代真实 AuthN/AuthZ。
- 审批同时绑定 request ID 与动作指纹；interrupt 前不做业务写入。最终动作只产生确定性 dry-run receipt，明确不声称外部副作用 exactly-once。
- 主线补充 `sync`、`async`、`exit` 的写入时机边界，并统一 LCEL 已在隔离 `langchain-core==1.4.9` 环境真实运行的证据。

## 2. CrewAI：持久状态与业务回执分层

- 新增 [[CrewAI/08-项目-真实CrewAI持久化Flow|项目：真实 CrewAI 持久化 Flow]] 和 `examples/crewai_layer_b/`，直接运行 `crewai==1.15.4`，不调用模型、不需要 API key。
- 示例验证 `Flow`、结构化 `FlowState`、`@start()`、`@router()`、`@listen()`、显式 `@persist(...)` 与 `SQLiteFlowPersistence`；路由标签与 listener 方法名分开，避免锁定版本中的自监听问题。
- Flow state UUID 只标识执行 lineage；业务 `operation_id` 与 `payload_hash` 负责幂等和冲突检测。Layer A 中原本容易混淆的确定性名称同步改为 `operation_id`。
- 独立 SQLite receipt ledger 与 Flow persistence 分工：测试在“receipt 已提交、Flow state 尚未保存”处注入故障，新进程用同一 UUID 水合后复用 receipt，串行 `effect_count` 保持 1。
- 同 UUID 恢复、从快照 fork、未知 UUID 失败关闭分别测试；正文明确 `@persist` 水合不是节点级 checkpoint 跳过，也不等于分布式 exactly-once。
- 匿名 telemetry、AMP tracing、Flow 生命周期事件与本地日志分别说明；AMP tracing 由每个 Flow 的 `tracing=False` 显式关闭，不把环境变量 `false` 误写成可靠 opt-out。Windows 测试在导入前启用 UTF-8，并只对锁定版本的两条完整传递依赖 warning 做窄范围 allowlist。

## 3. 学习路线元数据 v2：展示顺序与依赖关系解耦

- 新增 [[学习路线元数据v2规范]]，定义稳定课程 ID、知识域、全局目录顺序、角色无关硬前置，以及四条角色路线各自的顺序与 `core/recommended/optional` 类型。
- `.website/scripts/prepare-content.mjs` 改用固定 `yaml@2.9.0` AST 读取课程元数据。legacy 页面继续严格验证；页面一旦声明 schema 2，就不能缺字段或静默回退。
- 验证器检查 ID/domain/JavaScript 安全正整数类型、catalog order 唯一、硬前置存在/重复/自环/全图环、track order/kind 配对、角色与类型枚举、每个角色顺序唯一，以及非规范 YAML tag/anchor。
- 顶层课程索引若因发布策略退化为不含路线信息的 stub，构建直接失败；manifest 分开报告 legacy 与 v2 数量。
- 第一批只迁移 Tool Calling、Agent 核心、评测体系和 LLMOps，且只落两条已审硬边：`tool-calling → agent-core`、`evaluation → llmops`。没有把 MCP、环境型 Agent 或完整安全课程猜成普遍硬前置。
- 总路线补充框架的按需插入位置，并说明 legacy 展示顺序、角色推荐路径与真正硬依赖不是同一个概念。

## 核心来源

以下动态事实、版本与 API 在 2026-07-19 核对；课程只保留必要解释与直接链接：

- [LangGraph Interrupts](https://docs.langchain.com/oss/python/langgraph/interrupts)：interrupt 恢复和节点重放语义。
- [LangGraph Persistence](https://docs.langchain.com/oss/python/langgraph/persistence) 与 [Checkpointers](https://docs.langchain.com/oss/python/langgraph/checkpointers)：thread、checkpoint 与 durability 边界。
- [PyPI：langgraph 1.2.9](https://pypi.org/project/langgraph/1.2.9/)、[langgraph-checkpoint 4.1.1](https://pypi.org/project/langgraph-checkpoint/4.1.1/) 与 [SQLite checkpointer 3.1.0](https://pypi.org/project/langgraph-checkpoint-sqlite/3.1.0/)：隔离运行的 runtime 与序列化依赖快照。
- [CrewAI Flows](https://docs.crewai.com/en/concepts/flows) 与 [Checkpointing](https://docs.crewai.com/en/concepts/checkpointing)：Flow 装饰器、状态水合、同 UUID 与 fork 边界。
- [CrewAI Telemetry](https://docs.crewai.com/en/telemetry)、[Tracing](https://docs.crewai.com/en/observability/tracing) 与 [1.15.4 tracing enablement 源码](https://github.com/crewAIInc/crewAI/blob/1.15.4/lib/crewai/src/crewai/events/listeners/tracing/utils.py)：匿名遥测、AMP tracing 与锁定版本覆盖优先级。
- [PyPI：crewai 1.15.4](https://pypi.org/project/crewai/1.15.4/)：锁定运行版本和 Python 约束。
- [YAML Document API](https://eemeli.org/yaml/#documents) 与项目锁定的 `yaml@2.9.0`：课程 frontmatter 的 AST 解析和失败关闭验证。
