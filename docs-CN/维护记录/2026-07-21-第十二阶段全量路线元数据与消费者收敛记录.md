# 2026-07-21 第十二阶段全量路线元数据与消费者收敛记录

## 阶段结论

本阶段完成剩余 33 门顶层课程的逐页路线审阅，使 56/56 门课程全部进入学习路线元数据 v2；同时让 Obsidian 总路线、公开 Homepage、CourseNavigator 和站点验收器共同读取或校验同一套 `domain/catalog_order/hard_prerequisites/track` 关系。旧 `ai_learning_stage/order` 暂时保留用于历史兼容，但不再驱动三个主要路线消费者。

本阶段不是把 legacy order 机械改名。三个专项 Agent 分别审阅工程与数学基础、数据/框架、生产化与多模态课程；主 Agent 统一稳定 ID、domain、四角色顺序、`core/recommended/optional` 口径、跨组冲突与最终闭包。每个入口的“前置知识”都按完整课程级证据复核，没有把“建议先学”“会某个局部能力”或系统集成顺序擅自升级为硬依赖。

> [!important] 结论边界
> v2 路线能证明当前 56 个入口的目录、角色顺序和课程级硬前置在构建规则下自洽；它不证明所有 893 篇源 Markdown 都已逐段完成事实、来源、许可和示例深审。长期 `/goal` 继续保持 active。

## 33 门迁移与依赖判断

本阶段迁移以下课程：

- 工程与数学基础：AI 基础认知、Python、数据结构、API、Markdown、Git、Linux、正则、概率统计、线性代数、微积分、机器学习、深度学习；
- 数据、运行时与框架：向量基础、数据标注、数据可视化、MCP、环境型 Agent、Agent Skills、Agentic Design Patterns、LangChain、CrewAI；
- 生产化与多模态：MLOps、数据合成、隐私计算、多 Agent 协作、多模态 AI、OCR、语音识别、语音合成、实时多模态交互、图像生成、视频生成。

逐页证据只支持新增两组非空硬前置：

| 课程 | 硬前置 | 采用理由 |
| --- | --- | --- |
| `synthetic-data` | `data-cleaning`、`data-annotation`、`evaluation` | 入口明确写“先学习”三门完整课程；平台路线中三项均可见且更早 |
| `multi-agent-collaboration` | `agent-core` | 入口明确要求已完成 Agent 核心；工作流只是推荐顺序，具体框架可选 |

其余 31 门新迁移课程均声明 `hard_prerequisites: []`。完整硬依赖表见 [[维护记录/学习路线元数据v2规范|学习路线元数据 v2 规范]]；角色清单见 [[All of AI#机器可校验的角色路径|机器可校验的角色路径]]。

## 四条角色路径

| 角色 | 总数 | 核心 | 推荐 | 可选 | 主要取舍 |
| --- | ---: | ---: | ---: | ---: | --- |
| Agent 应用开发 | 31 | 9 | 13 | 9 | runtime、框架和多 Agent 均按需；完整安全在发布门附近 |
| RAG 与知识库 | 35 | 17 | 12 | 6 | 检索组件用 track 表达集成顺序；多模态摄取与 LangChain 是分支 |

受控 domain 仍有 10 个，当前 9 个包含课程；`frontier-reference` 暂为空。没有为了让每个 domain 都显示而创建没有明确学习产物的占位课程。

## 具体内容修正

- AI 基础认知进入四条角色路径的共同起点，解决课程自称“第一站”但旧角色摘要从模型选择起步的矛盾。
- Markdown 入口删除无语义空白表格列；正则表达式关系表改为可点击、文件夹限定的 wikilink。
- 机器学习与深度学习入口把完整课程从“隐含硬门”改为推荐背景或研究/多模态时按需学习。
- Agent Skills、Agentic Design Patterns、LangChain 和 CrewAI 统一区分“模块所需能力”“推荐顺序”与“完整课程硬前置”；Agent Skills 补回评测体系导航链接。
- CrewAI 保留经 Layer B 验证的 `crewai==1.15.4`，另记 2026-07-20 发布的最新观察 `1.15.5`；没有在未重跑测试时冒充基线升级。
- 多模态 AI 不再声称必须先完成 Embedding、Agent 与整套生产化；多 Agent 只保留 Agent 核心硬前置；视频生成把多门固定先修改为能力检查。
- MLOps 新增与 LLMOps 的职责交叉表：前者以训练数据、特征、训练代码和模型制品为发布对象，后者以模型供应方、Prompt、检索器、工具与策略组成的复合应用为发布对象。
- 生产化与多模态入口补充缺失的内容来源/状态字段，并修正本阶段涉及的项目标签大小写。

## 消费者收敛

`All of AI.md` 原先调用 `tools/dataview/ai-learning-roadmap`，但项目及上层 vault 中均不存在该 helper；Obsidian 源笔记实际会显示失败，而公开构建只是在 staging 时掩盖了问题。本阶段改为：

1. 源笔记直接保存静态知识域课程表和四条角色清单，Obsidian 无需额外 Dataview 代码即可阅读；
2. `prepare-content.mjs` 使用严格 YAML AST 重新生成同一快照并逐字比较，元数据变化而源路线未同步时构建失败；
3. 公开构建要求 56 门课程全部为 schema 2，任一课程退回 legacy 都会失败；
4. Homepage 按 v2 domain 展示课程，并从 track 计算四角色的核心/推荐/可选数量；
5. CourseNavigator 同时提供按角色排序的入口和按 domain 组织的完整课程树；
6. `validate-site.mjs` 改为校验 9 个已填充 domain、4 条角色路径、56 个课程入口和真实文件夹，不再寻找 legacy stage。
