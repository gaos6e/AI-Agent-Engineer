# JSON 学习目录

## 知识库简介

JSON（JavaScript Object Notation）是一种文本数据交换格式。API 请求与响应、配置文件、LLM 结构化输出、Tool Calling 参数、MCP 消息和 Agent 状态经常使用它。会辨认大括号只是起点：工程系统还必须处理 UTF-8、重复键、数值范围、解析资源上限、Schema 方言、版本迁移、错误脱敏，以及“结构正确但动作未获授权”的边界。

本库采用三个逐层收紧的视角：

1. **JSON 语法**回答“这段文本能否表示一个值”；
2. **数据契约**回答“这个值是否符合调用双方约定”；
3. **业务与授权**回答“这个值是否真实、可用，并允许触发动作”。

综合项目只处理本地教学数据，不访问网络、不使用密钥，也不会执行任何工具。它先用 Python 严格解析，再用 Draft 2020-12 Schema 验证，最后由可信代码判定工具建议是“仅验证”还是“需要审批”。

## 在总路线中的位置

本库位于 [[All of AI|AI Agent Engineer 总路线]] 的“工程与数学基础”知识域，并进入四条角色路径。它先建立请求体、响应体与配置的数据表示和契约；随后可进入 [[API/00-目录|API]]，再按需要学习 Markdown，并逐步进入 LLM、Tool Calling 与 MCP。

## 学习目标

- 写出并判断六类 JSON 值，区分 JSON 文本、Python 对象与“装着 JSON 的字符串”。
- 使用 `load/loads/dump/dumps`，正确处理 UTF-8、中文、错误行列号和序列化策略。
- 解释 Python 默认 `json` 与 RFC 8259 严格互操作边界，并拒绝重复键、非有限数字、孤立 surrogate 和超限输入。
- 区分单个 JSON 文档、JSON Lines、NDJSON 与 RFC 7464 JSON Text Sequences。
- 使用 Draft 2020-12 的核心关键字，理解 `$schema`、`$defs/$ref`、组合、条件、`default` 和 `format` 的真实语义。
- 为 Agent 配置与工具参数设计版本化契约、稳定错误路径和脱敏报告。
- 说明结构化输出为何仍不能证明事实、权限、审批或工具执行成功。
- 在普通与 `python -O` 模式下运行独立测试，验证失败路径而不是依赖裸 `assert`。

## 推荐学习顺序

1. [[JSON/01-语法与数据模型|语法与数据模型]]：从一个 JSON 文本走到六类值、空值语义与合法性判断。
2. [[JSON/02-Python解析、序列化与严格模式|Python 解析、序列化与严格模式]]：掌握标准库，并认识默认行为并不等于严格 RFC profile。
3. [[JSON/03-互操作性、Unicode与数字边界|互操作性、Unicode 与数字边界]]：处理重复键、对象顺序、Unicode、数值精度和 canonicalization。
4. [[JSON/04-文件、JSON Lines与流式处理|文件、JSON Lines 与流式处理]]：安全读写文件、逐行处理记录并理解原子替换的能力边界。
5. [[JSON/05-JSON Schema基础契约|JSON Schema 基础契约]]：用 Draft 2020-12 表达对象、数组、必填、范围与未知字段策略。
6. [[JSON/06-Schema设计、版本与错误定位|Schema 设计、版本与错误定位]]：学习复用、组合、条件、版本迁移与 RFC 6901 错误路径。
7. [[JSON/07-API、LLM与工具调用中的JSON|API、LLM 与工具调用中的 JSON]]：把格式、Schema、事实、授权和执行结果分层。
8. [[JSON/08-实战-可靠Agent配置与事件管道|实战：可靠 Agent 配置与事件管道]]：运行真实 Schema、严格 JSONL 管线、审批分流与脱敏报告。
9. [[JSON/09-练习、自测与掌握标准|练习、自测与掌握标准]]：完成故障修复、Schema 设计和项目扩展，再用清单验收。

## 动手实践入口

项目位于 `examples/`：

| 文件 | 用途 |
| --- | --- |
| `strict_json.py` | UTF-8 严格解析、唯一键、有限数字、资源限制、JSONL 与原子替换。 |

| `validate_agent_config.py` | 依次执行语法、Schema 和业务不变量验证。 |
| `tool_call.schema.json` | 两类本地工具建议的联合 Schema。 |

| `demo.py` | 在临时目录中完成读、验、原子写、读回闭环。 |
| `test_*.py` | 普通与 `-O` 模式均有效的 `unittest`。 |

## 掌握标准

- [ ] 能不运行代码判断常见 JSON 语法错误，并解释顶层标量为何合法。
- [ ] 能区分字段缺失、字段为 `null`、空字符串和空数组的业务含义。
- [ ] 能写出 UTF-8 读写代码，并解释 `ensure_ascii=False`、`allow_nan=False` 和双重编码。
- [ ] 能证明 Python 默认解析器会接受哪些非标准扩展，并按契约收紧。
- [ ] 能为重复键、超大输入、深层嵌套、JSONL 损坏行设计失败测试。
- [ ] 能读懂并修改 Draft 2020-12 Schema，知道验证器支持的方言与子集必须核对。
- [ ] 能用 JSON Pointer 定位错误，同时不把完整载荷或敏感值写入日志。
- [ ] 能说明 Schema 通过后仍需业务校验、可信工具注册表、授权、审批、幂等和结果确认。
- [ ] 能运行项目及全部测试，并独立增加一个字段、一个失败样例和一条迁移规则。
