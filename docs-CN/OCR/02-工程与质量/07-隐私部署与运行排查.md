# 隐私部署与运行排查

## 本节目标

把 OCR 看作处理原始文档和派生文本的数据系统，设计最小化、访问控制、日志、版本回滚与故障排查。

## 数据边界先于部署形态

本地部署不自动等于安全，云服务也不自动等于不合规。先画数据流：原件从哪里来、发送到哪里、产生哪些裁剪图/缓存/日志、谁能访问、保留多久。按照最小化原则，只传任务需要的页面或区域；测试数据应匿名化且有使用许可。

应区分：

- **原始文档**：可能包含身份证件、合同、医疗或财务信息。
- **派生图像**：裁剪区域仍可能泄露身份。
- **识别文本**：更易搜索和复制，泄露影响不比原图小。
- **调试日志**：不得默认记录全文、Base64 或永久下载链接。

来源撤权、用户删除请求或保留期到达时，应以 document/source revision 生成可审计事件，先阻断在线 OCR 结果、chunk、Embedding 与检索缓存，再逐层确认派生图、异步队列、评测副本和受控追踪的处理状态。物理清除、备份和法定保留可能不同步；不要把“已经从在线搜索移除”写成“所有副本已擦除”。具体传播与墓碑模式见 [[知识库构建/03-版本删除与权限|版本、删除与权限]]。

## 可运行的部署契约

锁定引擎、模型、语言包和预处理版本；启动时记录不含秘密的版本信息。升级采用影子评测或小流量切换，并保留回滚。容量计划至少测页面大小、每页耗时、峰值内存、并发和超时；超大页、加密 PDF、损坏图片要有明确拒绝路径。

## 故障定位

| 现象 | 首查 | 避免的误判 |
| --- | --- | --- |
| 整页空白 | 解码、方向、透明通道、检测框 | 立即更换识别模型 |
| 某语言全错 | 语言包、脚本检测、字体 | 用后处理强行替换 |
| 表格列错位 | 版面/表格结构输出 | 只优化字符 CER |
| 升级后分数突变 | 分数定义、阈值、适配层 | 认为数据突然变差 |
| 延迟偶发很高 | 页尺寸、队列、冷启动、重试 | 无上限地增加重试 |

日志使用文档 ID、页码、阶段、耗时、错误码和版本；敏感内容只在授权的短期诊断流程中最小化展示。

把 OCR 文本或截图交给 LLM/Agent 前，也要把它们视作不可信数据：页面中的“忽略规则”“导出附件”等文字不能改变系统指令、检索范围或工具权限。相关攻击与控制见 [[AI安全/00-目录|AI 安全]]；实际授权仍应在工具/数据服务端执行。

## 练习与自测

- 画出“上传扫描件—识别—人工复核—知识库”的数据保留表。
- 设计一个不记录正文仍可定位超时的日志事件。
- 回答：加密传输能否替代访问控制和删除策略？不能，它只解决传输链路的一部分风险。
