# 数据可视化

## 知识库简介

数据可视化把数据结构、模型误差与系统权衡变成人能检查的证据。它不是“给报告加一张漂亮图”，而是先明确论证问题和数据单元，再选择能暴露分布、不确定性与失败模式的视觉编码。Agent Engineer 常用它检查数据分布、分类错误、RAG 指标、成本、延迟和版本退化。

## 在总路线中的位置

本库位于“数学与数据基础”的末段，连接概率统计、机器学习、数据清洗、评测体系和运行监控。先会 Python 列表/字典与基本统计即可。

## 学习目标

- 从问题和读者行动出发选择图，而非从图形库函数出发。
- 正确展示分布、关系、时间变化、误差和不确定性。
- 为分类、检索、Agent 运行和成本评测设计组合图。
- 识别截断坐标、错误聚合、筛选偏差和颜色误导。
- 用 Matplotlib 生成可复现、带标签和来源的静态图。

## 前置知识

会在 PowerShell 7 创建 `venv`、安装包、运行 Python 3。理解均值、中位数、分位数和样本数；不熟悉时先学习[[Python基础/00-目录|Python 基础]]、[[概率统计/00-目录|概率统计]]与[[数据清洗/00-目录|数据清洗]]。本库示例使用虚构聚合数据，不需要联网或 API key。

## 推荐学习顺序

1. [[数据可视化/01-从问题到图表|从问题到图表]]：建立“问题—数据—编码—行动”选择框架。
2. [[数据可视化/02-分布与分组比较|分布与分组比较]]：避免均值掩盖长尾和子群。
3. [[数据可视化/03-关系趋势与时间序列|关系、趋势与时间序列]]：区分相关、变化和聚合造成的错觉。
4. [[数据可视化/04-误差与不确定性|误差与不确定性]]：让误差条、置信区间和多次运行可解释。
5. [[数据可视化/05-机器学习RAG与Agent评测图|机器学习、RAG 与 Agent 评测图]]：把组件指标与系统权衡放在一起。
6. [[数据可视化/06-误导风险与可访问性|误导风险与可访问性]]：审计坐标、颜色、筛选、标签和来源。
7. [[数据可视化/07-项目-Agent评测看板|项目：Agent 评测看板]]：生成四面板离线评测图并完成解读。

## 动手实践或项目入口

[[数据可视化/examples/agent_eval_dashboard.py|agent_eval_dashboard.py]]读取严格校验的[[数据可视化/examples/sample_agent_eval.json|虚构评测数据]]，绘制成功率 Wilson 95% 区间、尾延迟与超时、路由混淆矩阵、成本—成功率 Pareto 候选，并导出 PNG、SVG 与文字替代说明。[[数据可视化/examples/test_agent_eval_dashboard.py|自动测试]]覆盖数据契约、统计量、图结构、两种导出和 CLI；依赖版本记录在[[数据可视化/examples/requirements.txt|requirements.txt]]。

## 掌握标准

- [ ] 给定分析问题，能说明为什么选择某种图和编码。
- [ ] 展示均值时同时考虑分布、样本数和不确定性。
- [ ] 能读出混淆矩阵中的主要错误，而不是只报 accuracy。
- [ ] 能审查坐标范围、分箱、聚合、颜色和缺失数据处理。
- [ ] 能生成标题、轴标签、单位、图例、来源和可复现代码齐全的图。
- [ ] 能在最终尺寸下检查缺字、裁切、重叠、灰度可辨性与替代文本。
- [ ] 能运行普通与 `-O` 模式测试，并解释 Wilson 区间、行归一化混淆矩阵和 Pareto 候选。

## 与其他知识库的关系
