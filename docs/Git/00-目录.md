# Git

## 知识库简介

本目录根据 Git 官方 cheat sheet 整理，并补充了本机安装、每类命令的使用场景、常见风险和更现代的替代写法。命令本身保留英文原样，说明使用中文。

Git 是分布式版本控制系统。本库保留现有命令教程，并用一条面向 AI Agent Engineer 的从零路线把它们串起来：先建立“快照、对象、引用”的心智模型，再理解工作区、暂存区与 `HEAD`，之后学习分支、差异、撤销、远程协作，最后在隔离实验仓库中完成项目。不要在尚未理解影响时对真实工作区运行清理、硬重置、历史改写或强制推送。

> [!info] 来源
> 原始页面：[Git Cheat Sheet](https://git-scm.com/cheat-sheet)；PDF 版本：[cheat-sheet.pdf](https://git-scm.com/cheat-sheet.pdf)；Windows 安装页：[Installing Git](https://git-scm.com/install/windows)。

> [!important] 命令中的占位符
> 正文标题或行内命令里的 `<path>`、`<commit>`、`<branch>`、`<url>` 只表示“换成真实值”，不要把尖括号原样输入 PowerShell。为避免误复制，本库的 PowerShell 代码块使用 `README.md`、`HEAD~1`、`feature/demo` 等可解析示例；路径前的 `--` 用于明确结束选项并开始路径参数。

## 在总路线中的位置

本库位于工程基础阶段。它承接 Markdown 与命令行基础，为后续 Python 项目、Agent 工具、评测资产和生产配置提供可审查、可恢复的变更历史；Git 只追踪版本，不替代测试、制品存储、秘密管理或部署审批。

## 学习目标

- 解释工作区、暂存区、提交和分支之间的关系。
- 在提交前检查状态、差异、未跟踪文件与敏感信息。
- 使用分支隔离 Agent 代码、提示词、配置和评测变更。
- 选择 `restore`、`revert`、`reset`、`reflog` 等不同恢复方式。
- 安全同步远程仓库，并理解改写已共享历史的风险。
- 在隔离仓库完成从初始化到冲突解决、回退和复盘的实践。

## 动手实践与项目入口

综合任务见 [[Git/12-版本控制实战与自测|版本控制实战与自测]]。所有破坏性练习都应在新建的隔离实验仓库进行；先用 `git status`、`git diff` 和 `git log` 保存证据，再执行恢复命令。

## 掌握标准

- [ ] 能画出工作区→暂存区→提交的流向，并用命令验证。
- [ ] 每次提交前检查已暂存与未暂存差异、未跟踪文件和凭据风险。
- [ ] 能创建分支、制造并解决一个文本冲突，解释最终内容。
- [ ] 能区分撤销未提交改动、创建反向提交和移动分支指针。
- [ ] 能通过 `reflog` 找回可达性丢失的本地提交。
- [ ] 不对共享分支随意改写历史；必须强推时优先理解并使用 `--force-with-lease`。
- [ ] 能说明 `.gitignore` 为什么不能保护已进入历史的凭据。
