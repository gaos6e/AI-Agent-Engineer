## **1. SGD（随机梯度下降）**

### **1.1 原理（Plain SGD）**

**最基础的优化器，每次根据梯度往反方向走一步：**

```
$$w_{t+1} = w_t - \eta \cdot \nabla L(w_t)$$
```

- **优点：简单、理论成熟**
- **缺点：收敛慢，对 learning rate 很敏感**

### **1.2 PyTorch 调用示例**

```
import torch
model = ...  # 你的神经网络
optimizer = torch.optim.SGD(
model.parameters(),
lr=0.1,          # 学习率
)
```

### **1.3 常用参数说明**

- `lr`**：learning rate 学习率，步子大小**
    - **太大：训练震荡甚至发散**
    - **太小：收敛很慢**
- `weight_decay`**（可选）：L2 正则，防止过拟合**
    - **对应公式里加上 $\lambda \|w\|^2$**

## **2. SGD + Momentum（动量）**

### **2.1 原理**

**在 SGD 的基础上增加“惯性”，类似小球在山谷里滚：**

```
$$v_{t+1} = \mu v_t - \eta \nabla L(w_t)$$
```

```
$$w_{t+1} = w_t + v_{t+1}$$
```

- **$\mu$ 是动量系数（momentum）**
- **能在陡峭方向稳定下来，在缓慢方向加速收敛**

### **2.2 调用示例**

```
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.1,
    momentum=0.9,       # 动量系数
    weight_decay=1e-4,  # L2 正则
    nesterov=False      # 是否使用 Nesterov (下面讲)
)
```

### **2.3 重要参数**

- `momentum`**：动量系数 $\mu$**
    - **一般 0.9、0.95、0.99 这样**
- `nesterov`**：是否启用 Nesterov 动量**
    - `False`**：普通 Momentum**
    - `True`**：Nesterov Accelerated Gradient（见下一节）**

## **3. Nesterov Momentum（NAG）**

### **3.1 原理**

**和普通 Momentum 类似，但梯度是在“预估位置”上算的：**

- **普通动量：先算梯度再加动量**
- **Nesterov：先用动量“看一眼”前方，再在前方位置算梯度**

**在实践中：比普通 Momentum 稍微稳一些，但区别没那么巨大。**

### **3.2 调用示例**

```
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9,
    nesterov=True
)
```

> **参数同 SGD + Momentum，多了一个**`nesterov=True`**。**

## **4. AdaGrad**

### **4.1 原理**

**对每个参数记录历史梯度平方和，梯度累积越大，学习率就越小：**

```
$$G_t = G_{t-1} + g_t^2$$
```

```
$$w_{t+1} = w_t - \frac{\eta}{\sqrt{G_t} + \epsilon} g_t$$
```

- **对稀疏特征非常友好（NLP 早期常用）**
- **缺点：$G_t$ 会持续增大，学习率不断变小，后期几乎不动**

### **4.2 PyTorch 调用示例**

```
optimizer = torch.optim.Adagrad(
    model.parameters(),
    lr=0.01,
    lr_decay=0,    # 全局学习率衰减
    weight_decay=0,
    eps=1e-10
)
```

### **4.3 参数说明**

- `lr`**：初始学习率**
- `lr_decay`**：对**`lr`**做额外衰减，一般保持 0**
- `eps`**：防止除零的平滑项**
- `weight_decay`**：L2 正则**

## **5. RMSProp**

### **5.1 原理**

**为了解决 AdaGrad 学习率衰减过快的问题，改成指数滑动平均：**

```
$$E[g^2]_t = \alpha E[g^2]_{t-1} + (1-\alpha) g_t^2$$
```

```
$$w_{t+1} = w_t - \frac{\eta}{\sqrt{E[g^2]_t + \epsilon}} g_t$$
```

- **适合 RNN / 序列任务（Hinton 常用）**

### **5.2 调用示例**

```
optimizer = torch.optim.RMSprop(
    model.parameters(),
    lr=0.001,
    alpha=0.99,      # 梯度平方的 EMA 衰减系数
    eps=1e-8,
    weight_decay=0,
    momentum=0.0,
    centered=False   # 是否用 E[g] 进行居中
)
```

### **5.3 参数说明**

- `alpha`**：**`E[g^2]`**的衰减系数，类似“记忆长度”**
- `centered`**：**
    - `False`**（默认）：用 $\sqrt{E[g^2]}$**
    - `True`**：用 $\sqrt{E[g^2] - (E[g])^2}$，效果有时更稳**
- `momentum`**：也可以给 RMSProp 加动量**

## **6. Adam（目前最主流之一）**

### **6.1 原理**

**Momentum + RMSProp 的合体：**

- **一阶矩：梯度的指数滑动平均 $m_t$（类似动量）**
- **二阶矩：梯度平方的指数滑动平均 $v_t$（类似 RMSProp）**
- **再做偏差修正，最后更新参数**

**直观感受：**

- **自适应学习率 + 动量**
- **收敛快，对初学者比较“省心”**

### **6.2 调用示例**

```
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1e-3,
    betas=(0.9, 0.999),
    eps=1e-8,
    weight_decay=0
)
```

### **6.3 参数说明**

- `lr`**：整体学习率（Adam 默认 1e-3，比 SGD 常用的小）**
- `betas=(beta1, beta2)`**：**
    - `beta1`**：一阶矩（梯度均值）的衰减系数，默认 0.9**
    - `beta2`**：二阶矩（梯度平方均值）的衰减系数，默认 0.999**
    - **越接近 1，记忆越长、更新越平滑**
- `eps`**：数值稳定用的小常数**
- `weight_decay`**：L2 正则（注意：PyTorch 这个是“L2 正则”，而不是 AdamW 的 decoupled weight decay）**

---

## **7. AdamW（推荐在很多任务中用）**

### **7.1 原理**

**Adam 的一个改进版本：把 weight decay 从梯度更新里“分离”出来，理论上更合理，在 Transformer / BERT 等模型中几乎是标配。**

**更新大致变为：**

- **Adam 部分更新一次**
- **再做一个： $w \leftarrow w (1 - \eta \cdot \lambda)$**

### **7.2 调用示例**

```
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=5e-4,
    betas=(0.9, 0.999),
    eps=1e-8,
    weight_decay=0.01
)
```

### **7.3 参数说明**

**和 Adam 非常类似，多数参数同名：**

- `lr`**：推荐比 Adam 稍微小一点点视任务而定**
- `betas`**/**`eps`**：同 Adam**
- `weight_decay`**：真正意义上的权重衰减（decoupled），常用 0.01、0.05 等**

---

## **8. 选型建议：什么时候用哪个？**

**粗略经验（不绝对）：**

1. **小模型 / 传统任务（比如简单 CNN、MLP）**
     - **起步可用**`SGD + momentum`
     - **如果调参经验少，**`Adam`**/**`AdamW`**也完全 OK**
2. **大模型 / NLP / Transformer 系列**
     - **绝大多数情况使用**`AdamW`
     - **配合合适的学习率调度器（warmup + cosine 等）**
3. **RNN、序列任务（早期）**
     - `RMSProp`**有时不错，但现在基本也直接用 Adam/AdamW 了**
4. **有大量稀疏特征（非常高维的 one-hot）**
     - `AdaGrad`**在某些场景仍有用**
     - **但业界还是大多统一到 Adam/AdamW**
