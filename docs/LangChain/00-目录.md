# LangChain

## 知识库简介

本知识库分为两层：先用 `00-初学者路线/` 从零建立组件、LCEL、模型与消息、工具、检索、状态、LangGraph 和测试的完整工程路线；再按需查阅官方文档译文与案例。框架 API 变化快，学习重点是稳定的输入输出契约和控制边界，而不是背诵某个版本的导入路径。

既有文件是按固定日期获取的官方文档译文与参考层；`00-初学者路线/` 是面向从零学习者的新讲解层。两层分开，便于保留来源和许可，也便于在依赖升级时只复核代码边界。

> [!info] 当前版本基线
> 2026-07-14 核对时，PyPI 当前发布为 `langchain 1.3.13`（2026-07-10）、`langgraph 1.2.9`（2026-07-10）与 `langchain-core 1.4.9`（2026-07-08）。这些数字只描述本次资料快照，不表示三个包可以脱离解析器任意组合；新项目应在隔离环境中安装、生成锁文件并运行测试。既有官方译文获取于 2026-05-07，可能早于当前 API，适合作为参考层而非无需复核的复制粘贴来源。

## 在总路线中的位置

本库位于“框架实践”阶段。先理解模型、提示、工具、检索和 Agent 状态，再用 LangChain 快速装配常见应用；当流程需要显式分支、持久状态、暂停恢复或精细编排时，再下沉到 LangGraph。

## 学习目标

- 理解 LangChain、LangGraph、`langchain-core` Runnables（LCEL）与 Deep Agents 的职责边界。
- 能用 LCEL 组合确定性步骤，并判断普通 Python、Runnable、Agent 与状态图各自适用的场景。
- 能用消息、模型、工具和结构化输出描述一个最小 Agent 循环。
- 能把检索器接入生成流程，并区分短期状态与长期记忆。
- 能判断何时需要 LangGraph 的检查点、interrupt 和显式状态图。
- 能为工具权限、失败恢复、离线评测和生产观测设计边界。

## 前置知识

- Python 函数、类型提示、虚拟环境和异常处理。
- HTTP/JSON 与 LLM API 的基本概念；不要求已有 LangChain 经验。
- 涉及真实模型时需自行选择提供商并使用环境变量；本路线不包含真实密钥。

## 动手实践或项目入口

- 每篇初学者笔记都有设计练习；综合入口是 [[LangChain/00-初学者路线/07-项目-离线工具代理骨架|离线工具代理骨架]]。
- `00-初学者路线/examples/offline_agent_loop.py` 只用 Python 标准库，模拟“模型提出工具调用→执行器校验→工具返回→模型生成最终回答”的循环。
- `00-初学者路线/examples/lcel_no_key.py` 是不需要模型密钥的 LCEL 组合示例；本机未安装 `langchain-core`，因此只做了语法检查，按项目锁定依赖后再运行。
- `00-初学者路线/examples/test_offline_agent_loop.py` 对离线循环执行确定性单元测试，不访问网络。
- 真实 LangChain 示例需在独立 `venv` 中安装并锁定项目版本；未配置依赖和模型凭据时，不应声称端到端已运行。

## 掌握标准

- [ ] 能解释 `create_agent` 适合的高层场景，以及何时改用 LangGraph。
- [ ] 能解释 LCEL 的 `RunnableSequence`、`RunnableParallel` 与 `invoke/batch/stream` 契约，不把旧版 `LLMChain` 教程当作 v1 首选入口。
- [ ] 能为工具定义清晰输入、错误和权限边界。
- [ ] 能区分消息历史、运行状态、检查点和长期记忆。
- [ ] 能设计一个带检索、引用、人工审批、恢复和离线评测的最小应用。
