# 主动学习与人机协同

## 本节目标

设计“模型选样—人独立判断—复审—再训练”的闭环，同时保留随机对照、冻结评测集和预标注盲测，避免效率提升掩盖分布偏差。

## 主动学习的目标

当未标注池很大时，主动学习让模型选择“最值得人标”的样本，以更少标签改善模型。常见候选策略：

- **不确定性**：模型最拿不准的样本。
- **多样性/代表性**：覆盖不同区域，避免只标同一种难例。
- **预期价值**：优先高风险或高业务影响样本。

只选不确定样本会过度集中在噪声、离群点和标签本来就含糊的区域。因此实践中应混合随机代表样本、风险样本与不确定样本，并分别报告表现。

主动选出的样本不代表线上自然分布，不能直接用其标签比例估计总体发生率或整体准确率。应保留独立随机审计流；若要做总体估计，需要记录采样概率并采用与设计匹配的估计方法。

## 闭环

1. 用已标数据训练当前模型。
2. 在未标池生成候选分数，不接触封存评测集。
3. 去重、分层并抽取一批样本。
4. 人独立标注与复审，记录候选原因。
5. 加入训练集、升数据版本、重新评估。
6. 比较随机抽样基线，确认收益来自策略而非样本数量。

## 模型辅助标注的锚定风险

预标注能减少重复操作，但人看到模型答案后更可能接受它，尤其在难例上。控制方法：

- 用一部分盲标样本估计预标注偏差。
- 隐藏置信度或延迟显示建议。
- 对模型高置信与低置信样本都做随机审计。
- 保留“模型建议、人工初标、复审、最终标签”四层，不相互覆盖。

## Agent 场景的人机边界

模型可做去重候选、风险排序和格式检查；人负责规则外异常、价值判断、伤害权衡与最终裁决。高风险标签不能因“模型很自信”而取消人工复核。

## 练习

为 10 万条 Agent 轨迹设计每轮 500 条的采样配方，明确随机、风险、不确定和新版本样本占比；写出如何防止测试集混入训练闭环。

## 掌握检查

- [ ] 我会把随机、风险、不确定、多样性和新版本样本分层记录。
- [ ] 主动学习池永不读取冻结评测集，模型版本与候选原因可追溯。
- [ ] 我能用盲标子集估计预标注锚定，而不是只看标注速度和一致率。
- [ ] 我不会用主动抽样批次的标签比例直接声称线上总体发生率。
