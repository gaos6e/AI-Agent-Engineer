# 数据标注

## 知识库简介

数据标注把模糊的业务判断变成可重复的数据规则。标签不仅用于监督训练，也用于 RAG 相关性判断、Agent 轨迹评审、安全红队与离线评测。真正的交付物是“任务定义 + 指南 + 标注结果 + 质量证据 + 版本记录”，不是一列看似完整的标签。

## 在总路线中的位置

本库位于“数学与数据基础”，应在基本数据清洗之后学习，并为机器学习、Reranking、RAG、评测体系和数据合成提供可靠监督信号。

## 学习目标

- 定义标注单元、标签空间、排除条件和证据范围。
- 写出初学标注者可以执行的指南与边界案例。
- 通过试标、重叠标注、盲审和裁决控制质量。
- 正确解释一致率与 Cohen's kappa，不把一致性当作正确性。
- 设计主动学习和模型辅助流程，同时控制自动化偏差。

## 前置知识

理解 JSONL/CSV 与抽样概念；会运行 Python 3 标准库脚本。建议先完成 [[数据清洗/00-目录|数据清洗]]、[[JSON/00-目录|JSON]]和[[概率统计/00-目录|概率统计]]的抽样与不确定性部分。标注数据可能包含个人信息，开始前应确定最小访问权限、脱敏和保留期限。

## 推荐学习顺序

1. [[数据标注/01-任务定义与标注单元|任务定义与标注单元]]：把“好不好”改成可观察、可裁决的标签。
2. [[数据标注/02-标注指南与边界案例|标注指南与边界案例]]：用决策规则、正反例和弃权机制减少歧义。
3. [[数据标注/03-标注流程与数据格式|标注流程与数据格式]]：建立可追踪的人员、样本、工具和版本流程。
4. [[数据标注/04-质检复审与裁决|质检、复审与裁决]]：把错误发现和争议解决设计进生产线。
5. [[数据标注/05-一致性指标|一致性指标]]：手算一致率与 kappa，并理解局限。
6. [[数据标注/06-主动学习与人机协同|主动学习与人机协同]]：用模型提高效率，但保留独立判断和代表性抽样。
7. [[数据标注/07-项目-Agent回答质量标注|项目：Agent 回答质量标注]]：完成指南、双标、冲突审查和审计报告。

## 动手实践或项目入口

[[数据标注/examples/audit_annotations.py|audit_annotations.py]] 对 [[数据标注/examples/sample_annotations.jsonl|sample_annotations.jsonl]]执行严格 schema/版本/双人配对校验，计算百分比一致率、Cohen's kappa、混淆计数和冲突样本。[[数据标注/examples/test_audit_annotations.py|test_audit_annotations.py]]覆盖 10 类正常与失败行为；项目不需要第三方包或 API key。

## 掌握标准

- [ ] 能让两名未参与设计的人按指南独立试标，并收集歧义。
- [ ] 能区分“标签一致”“标签正确”和“数据代表性”。
- [ ] 能为多类别、序列标注或生成式评分选择合适的一致性/质量证据。
- [ ] 能追溯每个标签的样本版本、指南版本、标注者和时间。
- [ ] 能解释模型预标注可能怎样锚定人的判断。
- [ ] 能运行普通/`-O` 模式与 10 项测试，并解释常量标签完美一致时 kappa 为何未定义。

## 与其他知识库的关系

- [[数据清洗/00-目录|数据清洗]]负责在标注前发现损坏、重复和敏感字段。
- [[机器学习/00-目录|机器学习]]使用标签训练与评估，但不能突破标签定义本身的上限。
- [[数据可视化/00-目录|数据可视化]]用于检查类别分布、标注者偏差和冲突矩阵。
- [[评测体系/00-目录|评测体系]]与[[Benchmark设计/00-目录|Benchmark 设计]]会把标注指南转化为评分 rubric 和冻结评测集。
- [[RAG/00-目录|RAG]]与[[Reranking/00-目录|Reranking]]常需要查询—文档的分级相关性，而不是简单二元标签。
- [[数据合成/00-目录|数据合成]]可生成候选与边界样本，但合成来源、模型建议和人工最终标签必须分层保存。
