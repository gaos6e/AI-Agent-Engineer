# 提示词工程

> 资料核验日期：2026-07-21。提示策略会随模型、API 与安全机制演进；本库把可迁移的工程方法与供应商特性分开。涉及线上模型时，必须重新核对所用供应商的当前文档，并用自己的评测集验证。

## 知识库简介

提示词工程（prompt engineering）不是寻找一句“万能咒语”，而是把业务目标写成可测试的输入契约：说明任务、输入边界、约束、输出格式和成功标准，再用代表性样例验证。可迁移的方法包括任务定义、数据与指令分离、契约校验、版本管理和评测；角色名称、schema 支持范围、提示缓存或托管方式则可能因供应商与模型而异。提示词不能代替正确的数据、模型选择、检索、权限控制或程序校验。

## 在总路线中的位置

本库位于 LLM 应用基础阶段。学完基础的 API、JSON 与 Markdown 后，可先用 [[现代LLM能力与模型选择/00-目录|现代 LLM 能力与模型选择]] 写清任务能力与硬门槛，再在这里学习如何把意图表达清楚；随后在 [[上下文工程/00-目录|上下文工程]] 中组织证据与状态，在 [[LLM API集成/00-目录|LLM API集成]] 中把提示接入可靠程序。

## 学习目标

- 把模糊需求改写成包含成功标准的任务说明。
- 正确使用角色、分隔符、示例与反例，并能指出哪些做法是供应商特性。
- 用 JSON Schema 或等价契约约束机器可读输出，并在程序端继续验证语义。
- 版本化提示模板，以固定样例和指标驱动迭代。
- 识别提示注入，建立“指令与不可信数据分离”的信任边界。

## 前置知识

- 能阅读基本 Markdown、JSON 和 Python。
- 理解 HTTP 请求与响应的基本概念。
- 不要求机器学习数学基础，也不要求拥有任何模型 API 密钥。

## 推荐学习顺序

1. [[提示词工程/01-任务定义、指令层级与边界|任务定义、指令层级与边界]]：先定义成功，再写指令。
2. [[提示词工程/02-零样本、示例与反例|零样本、示例与反例]]：知道何时增加示例以及如何避免示例偏差。
3. [[提示词工程/03-结构化输出与契约|结构化输出与契约]]：让输出能被程序安全消费。
4. [[提示词工程/04-模板、变量与版本管理|模板、变量与版本管理]]：把一次性提示变成可维护资产。
5. [[提示词工程/05-评测驱动迭代|评测驱动迭代]]：用数据而不是印象决定是否改进。
6. [[提示词工程/06-提示注入与信任边界|提示注入与信任边界]]：把外部文本当数据，最小化可造成的影响。
7. [[提示词工程/07-提示词实验项目与自测|提示词实验项目与自测]]：完成一个离线分类契约实验。

## 动手实践与项目入口

从 [[提示词工程/07-提示词实验项目与自测|提示词实验项目]] 开始。项目由 [[提示词工程/examples/prompt_lab.py|离线实验脚本]]、[[提示词工程/examples/cases.json|12 条版本化案例]]、[[提示词工程/examples/response.schema.json|响应 schema]] 和 [[提示词工程/examples/test_prompt_lab.py|单元测试]] 组成，无需联网或密钥。它验证角色分离、严格 JSON、代码内提示与 schema 版本绑定、案例标注依据、合同漂移失败关闭、证据落地、资产哈希、退出码和分组统计；它不验证任何线上模型的真实质量。

## 掌握标准

- 能为一个真实任务写出输入、输出、约束和拒绝条件。
- 能解释“有效 JSON”和“符合 schema 且语义正确”的差别。
- 能用至少 10 个代表性样例比较两个提示版本，按切片和风险记录失败，而不是只看总平均。
- 能指出提示注入无法只靠一句防御文本解决，并给出权限、校验和人工确认措施。
- 能让模板改动、模型配置与评测结果彼此可追溯。
- 能运行本库 19 项离线测试，并让错误响应、畸形案例或合同漂移以约定的非零状态失败。

## 与其他知识库的关系

- [[现代LLM能力与模型选择/00-目录|现代 LLM 能力与模型选择]]先判断候选是否在本任务上具备所需推理、结构化输出、工具和模态能力；提示词不能补偿未通过的硬能力或隐私门槛。
- [[上下文工程/00-目录|上下文工程]]决定把哪些材料放入提示及其顺序与预算。
- [[LLM API集成/00-目录|LLM API集成]]负责角色映射、超时、重试、流式事件、用量和日志。
- [[RAG/00-目录|RAG]]提供可追溯证据，[[评测体系/00-目录|评测体系]]判断提示变化是否真正提升任务表现。
- [[AI安全/00-目录|AI安全]]与权限系统限制失败后的影响范围；[[Tool Calling（含 Function Calling）/00-目录|Tool Calling]]把经过验证的模型建议接到真实动作上。
