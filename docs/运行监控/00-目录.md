# 运行监控

## 知识库简介

运行监控让团队知道服务是否满足用户期望，并在延迟、错误、质量、安全或成本异常时快速定位、遏制和学习。**监控（monitoring）**通常针对预先定义的信号和条件；**可观测性（observability）**更强调通过系统输出回答事先未枚举的问题。两者都依赖有意义的仪器化、语义和运行流程。

本库聚焦Logs、Metrics、Traces、仪器化与上下文传播、SLI/SLO、质量/安全/成本信号、告警、Dashboard、漂移取证和事件响应。它不代替 [[MLOps/00-目录|MLOps]] 的模型生命周期或 [[LLMOps/00-目录|LLMOps]] 的LLM发布单元与评测门；也不重新定义 [[评测体系/00-目录|评测体系]] 中的评分器，而是监控这些线上信号的覆盖率、趋势和行动边界。

## 在总路线中的位置

运行监控位于“已部署”与“可靠运营”之间。建议先理解API、JSON、日志、模型/LLM调用链和基本评测，再学习本库。后续可将运行证据用于评测体系、Benchmark设计、AI安全和AI治理。

## 学习目标

完成后，你应能：

- 区分Log、Metric和Trace适合回答的问题，并用ID、资源属性和时间关联它们；
- 设计自动/手工仪器化、W3C Trace Context传播和Collector流水线，并监控遥测链自身；
- 从用户期望定义SLI、SLO、统计窗口和错误预算；
- 用RED、USE和黄金信号组织服务与资源症状，正确解释直方图、百分位、采样和高基数；
- 将可用性、尾延迟与AI任务质量、安全、费用和标签覆盖率组成监控面；
- 设计可操作、可抑制、有runbook和明确所有者的告警；
- 用概览—下钻—证据链设计Dashboard，避免为好看堆积图表；
- 正确解释漂移、标签延迟、采样和缺失信号；
- 执行事件分级、遏制、取证、恢复与复盘闭环。

## 前置知识

- 能用Python 3读写JSON，理解平均数、百分位、比率和时间窗口；
- 知道HTTP状态码、请求ID、超时、重试和批处理任务；
- 对模型版本、RAG、Tool Calling和LLM token/费用有概念；
- 不要求先安装Prometheus、OpenTelemetry或云监控平台。

## 推荐学习顺序

1. [[运行监控/01-可观测性基础/01-Logs、Metrics与Traces|Logs、Metrics 与 Traces]]：先理解三类信号的取舍与关联。
2. [[运行监控/01-可观测性基础/02-仪器化、Collector与关联|仪器化、Collector 与关联]]：让遥测跨HTTP、异步任务和工具边界仍可关联、可治理。
3. [[运行监控/01-可观测性基础/03-SLI、SLO与错误预算|SLI、SLO 与错误预算]]：从用户期望选信号和目标。
4. [[运行监控/02-生产监控/03-质量、安全与成本信号|质量、安全与成本信号]]：让“HTTP 200”之外的AI失败可见。
5. [[运行监控/02-生产监控/04-告警设计与值班运行|告警设计与值班运行]]：只将需要行动的症状送给正确的人。
6. [[运行监控/02-生产监控/05-Dashboard与诊断视图|Dashboard 与诊断视图]]：建立从用户影响到单次Trace的下钻路径。
7. [[运行监控/02-生产监控/06-漂移、反馈与标签延迟|漂移、反馈与标签延迟]]：将分布变化当作调查起点，而不是自动根因。
8. [[运行监控/02-生产监控/07-事件响应与复盘|事件响应与复盘]]：在压力下保护用户、保留证据并验证恢复。
9. [[运行监控/03-项目与自测/08-离线监控审计项目与自测|离线监控审计项目与自测]]：验证结构化事件、Trace、RED/USE、SLO、错误预算和多窗口告警。

## 动手实践与项目入口

主项目见 [[运行监控/03-项目与自测/08-离线监控审计项目与自测|离线监控审计项目与自测]]。它仅用Python标准库，严格验证小型JSON事件与Trace契约，计算RED/USE、SLI、p95、错误预算燃烧率、质量/安全/成本及Collector健康，并模拟正常窗口、工单窗口与多窗口PAGE。所有ID和阈值都是离线教学数据，不需要网络、第三方包或密钥。

## 掌握标准

- [ ] 能为一次慢请求从Metric下钻到Trace和结构化Log；
- [ ] 能解释W3C Trace Context、Collector流水线、语义约定稳定性、采样与基数的边界；
- [ ] 能写出包含事件定义、好事件条件、窗口和数据来源的SLI；
- [ ] 能区分技术成功、任务质量、安全结果、成本和观测完整性；
- [ ] 能让告警具有用户影响、严重级别、所有者、runbook、抑制和恢复条件；
- [ ] 能解释采样、高基数、标签延迟和代理指标造成的证据限制；
- [ ] 能从事件复盘产生新告警、回归样本、runbook和演练；
- [ ] 能在Windows 11/PowerShell 7中运行离线项目及32项测试，并说明为什么每个阈值只是本地教学策略。
