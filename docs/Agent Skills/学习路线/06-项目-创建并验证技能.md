# 项目：创建并验证 Agent Skill

## 项目目标

检查一个完整的 `text-statistics` Skill：合法 frontmatter、明确触发边界、无网络脚本、结构化输出和 eval cases。然后做一次小改动并用负向测试证明校验有效。

## 项目结构

```text
examples/
├── validate_skill.py                 # 严格教学 profile 验证器
├── test_validate_skill.py            # 51 个离线回归测试
└── text-statistics/
    ├── SKILL.md
    ├── scripts/
    │   └── text_stats.py
    └── evals/
        └── evals.json
```

样例只读取显式命令行文本或不超过 1 MB 的 UTF-8 文件，把统计结果写到 stdout；它不访问网络、不写业务文件、不需要凭据，也不回显原文。

## 在 PowerShell 7 运行

```powershell
Set-Location "X:\path\to\your-vault\Knowledge\AI Agent Engineer\docs\Agent Skills"
python -B .\examples\validate_skill.py .\examples\text-statistics
python -B .\examples\text-statistics\scripts\text_stats.py --text "Hello Agent 世界"
python -B .\examples\text-statistics\scripts\text_stats.py --help
python -B .\examples\test_validate_skill.py
python -B -O .\examples\test_validate_skill.py
```

预期：第一条输出 `status: ok`、正反各 10 条触发用例和已检查脚本的 JSON；第二条输出包含 `words`、`characters`、`lines`；第三条展示非交互参数说明；两次测试都应报告 51 项通过。`-B` 避免生成 `__pycache__`，`-O` 复跑可证明测试不依赖会被优化移除的裸 `assert`。

## 阅读顺序

1. 打开 [SKILL.md](Agent%20Skills/examples/text-statistics/SKILL.md)，找出“做什么”和“何时用”。
2. 检查 name 是否与父目录 `text-statistics` 完全一致。
3. 检查 body 何时让 agent 运行 [text_stats.py](Agent%20Skills/examples/text-statistics/scripts/text_stats.py)。
4. 阅读 [evals.json](Agent%20Skills/examples/text-statistics/evals/evals.json)，审查正例与近似负例是否真实；它是本课程触发语料格式，不是官方通用 schema。
5. 运行验证器和测试，再在临时副本中故意把 name 改为 `Text-Statistics`，确认校验失败；不要破坏课程原件。

## 扩展任务

新增 `--top-words N` 前先写 contract：输入范围、中文/英文 token 规则、大小写、标点和输出 schema。然后：

- 更新脚本与 `--help`；
- 只有范围确实改变时才更新 description；
- 新增正常、空文本、非法 N 三个测试；
- 检查默认输出没有泄露原始全文。

## 验收清单

- [ ] 本地教学验证器退出码为 0。
- [ ] 脚本无交互、无网络、stdout 为 JSON、缺参时错误可操作。
- [ ] description 含能力与使用时机，且不过度覆盖改写/翻译任务。
- [ ] evals 含正反各 10 条触发用例，负例覆盖相邻任务而非显然无关问题。
- [ ] 51 个回归测试在正常、`-O` 与 warnings-as-errors 三种模式通过。
- [ ] 能说明本地验证器不是完整 YAML 解析器，也不是官方规范认证。
- [ ] 若具备官方 `skills-ref`，另运行 `skills-ref validate ./examples/text-statistics`；未安装时如实记录未执行。

## 自测

1. 为什么父目录名也是契约的一部分？它让发现和 Skill identity 保持稳定。
2. 为什么 should-not-trigger case 不要求脚本输出？它检验的是 Skill 是否应被加载，而不是运行结果。
3. 如果脚本需要真实文件写入，应新增哪些保护？至少包括目标范围、dry-run/确认、幂等或回滚、脱敏日志。
