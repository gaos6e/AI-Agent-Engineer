## 在总路线中的位置

本库位于“单 Agent 与工具”阶段：先掌握 Tool Calling、Agent 核心状态循环，再用设计模式把路由、并行、反思、规划、记忆、人工审批、恢复和评测组合成可控系统。它回答的是“何时选哪种结构”，而不是绑定某个框架。

## 学习目标

- 能从确定性工作流、单 Agent 和多 Agent 中选择最低复杂度方案。
- 能画出状态、分支、工具权限、停止条件和失败路径。
- 能为高风险动作设置人工审批，为长任务设置检查点、重试与幂等键。
- 能用任务成功率、成本、时延、安全违规率和人工接管率评测系统。

## 前置知识

- 会读基础 Python 函数、JSON 状态和 HTTP 请求即可；不会并发编程也可以先理解图和状态。
- 建议先完成 [[Agent 核心/00-目录|Agent 核心]] 与 [[Tool Calling（含 Function Calling）/00-目录|Tool Calling（含 Function Calling）]]；不了解框架不影响本路线。

## 推荐学习顺序

1. [[Agentic Design Patterns/00-初学者路线/01-模式选择与最小架构|模式选择与最小架构]]：先判断是否真的需要 Agent。
2. [[Agentic Design Patterns/00-初学者路线/02-路由并行与汇聚|路由、并行与汇聚]]：掌握最常用的确定性组合。
3. [[Agentic Design Patterns/00-初学者路线/03-反思规划与停止条件|反思、规划与停止条件]]：让迭代有预算、有出口。
4. [[Agentic Design Patterns/00-初学者路线/04-工具记忆与状态边界|工具、记忆与状态边界]]：明确外部动作和长期状态的职责。
5. [[Agentic Design Patterns/00-初学者路线/05-人工审批与安全边界|人工审批与安全边界]]：把不可逆动作挡在授权门之后。
6. [[Agentic Design Patterns/00-初学者路线/06-失败恢复评测与可观测性|失败恢复、评测与可观测性]]：用证据判断系统是否可靠。
7. [[Agentic Design Patterns/00-初学者路线/07-项目-可恢复任务工作流|项目：可恢复任务工作流]]：完成一个无需密钥的离线项目。

## 掌握标准

- [ ] 面对新需求，能说明为何选工作流、单 Agent 或多 Agent。
- [ ] 能为循环写出最大步数、完成判据和失败判据。
- [ ] 能区分可重试故障、业务拒绝和需要人工介入的异常。
- [ ] 能设计离线回放集，并给出质量、成本、时延和安全门槛。
- [ ] 能证明审批绑定当前动作，并让重复恢复不重复副作用。
- [ ] 能同时检查轨迹和环境最终状态，而不是只看 Agent 的文字声明。

## 与其他知识库的关系

- 本库的模式应先于框架选型；迁移框架时，状态、权限、评测和恢复约束仍应保留。
