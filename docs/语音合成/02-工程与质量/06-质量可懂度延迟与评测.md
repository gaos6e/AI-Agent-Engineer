# 质量、可懂度、延迟与评测

## 本节目标

把“听起来不错”拆成可懂度、自然度、内容正确性、说话人一致性、稳定性和系统延迟。

## 主观评测

MOS（Mean Opinion Score，平均意见分）通常让听众在有定义的等级上评分，再取平均。它对实验说明、样本、播放设备、听众语言和量表非常敏感；不同实验的 MOS 不能脱离设计直接横比。成对偏好（A/B）让听众选择更偏好的版本，适合小差异，但仍需随机顺序和足够样本。

至少把问题分开：

- **自然度**：像自然语音吗？
- **可懂度**：内容能否正确听出？
- **内容正确性**：数字、人名、否定和单位是否读对？
- **适配性**：语气是否符合场景，而非是否喜欢某种声音。

涉及可识别声音时，听众评价和样本展示也必须在授权范围内。

## 客观与系统指标

可以用 ASR 回转测可懂度代理：合成后再识别并算 WER，但结果同时受 ASR 偏差影响，不能替代人听。还应测发音词典命中、数字/日期 exact match、静音比例、削波、音频时长异常和失败率。

系统侧记录 p50/p95/p99 首包延迟、总生成延迟、实时率、超时、取消成功率和缓存命中。把短句与长句、语言、声音、流式/批处理切片。

## 实验设计

冻结文本集、声音、配置、设备与版本；样本顺序随机、音量归一且不泄露系统标签。报告听众数、每项样本数、评分说明和不确定性。不要只挑“演示最好”的句子，也要包含数字、缩写、多语言和长句。

## 练习与自测

1. 设计 20 条覆盖数字、姓名、列表、疑问和警告的评测文本。
2. 为什么 ASR 回转 WER 低不等于自然？机器人声音也可能很清楚。
3. p95 首包延迟改善但取消失败率上升，应否直接上线？不能，只看一个指标不足。
