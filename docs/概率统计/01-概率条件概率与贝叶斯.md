# 概率、条件概率与贝叶斯

## 为什么需要

Agent 的工具可能失败，分类器可能误报，检索结果可能相关。概率不是“主观含糊”，而是用 0 到 1 描述不确定事件的模型。

## 事件与基本规则

样本空间 $\Omega$ 是所有可能结果，事件 $A$ 是其中一部分。概率满足 $0\le P(A)\le1$、$P(\Omega)=1$。补集规则：

$$P(\neg A)=1-P(A)$$

若两事件互斥，$P(A\cup B)=P(A)+P(B)$；一般情况需减去重复部分：

$$P(A\cup B)=P(A)+P(B)-P(A\cap B)$$

例：100 次工具调用有 12 次超时，经验超时率为 $12/100=0.12$。这是样本估计，不是永恒真值。

## 条件概率与独立

知道 $B$ 已发生后，$A$ 的概率为：

$$P(A\mid B)=\frac{P(A\cap B)}{P(B)},\quad P(B)>0$$

若 $P(A\mid B)=P(A)$，才称 $A$ 与 $B$ 独立。不同请求来自同一用户、同一服务故障窗口或同一文档时，往往不独立；把它们当独立样本会低估不确定性。

例：监控显示总体失败率 5%，高峰时段失败率 15%。你在高峰已知条件下应使用 15%，不能仍用总体 5%。

## 贝叶斯公式

$$P(H\mid E)=\frac{P(E\mid H)P(H)}{P(E)}$$

$H$ 是假设，$E$ 是证据。$P(H)$ 是先验，$P(E\mid H)$ 是证据在假设成立时出现的概率，$P(H\mid E)$ 是更新后的后验。

假设线上只有 1% 请求真正异常；告警器对异常的召回率 90%，对正常请求误报率 5%。告警后真正异常概率：

$$\frac{0.90\times0.01}{0.90\times0.01+0.05\times0.99}\approx0.154$$

即约 15.4%，不是 90%。低基率让误报占多数。这对安全检测、幻觉告警和异常监控非常重要。

## ML、检索与评测中的用途

- 分类模型输出的“概率”需校准后才可按频率解释；softmax 数值不自动保证可信。
- RAG 中“检索到相关文档”会改变回答正确的条件概率，但两者不是同一事件。
- 离线评测按任务类别分层，是在不同条件下比较成功率。
- 多步 Agent 成功率不能盲目相乘，除非步骤失败近似独立。

## 常见误区

- 把 $P(A\mid B)$ 当成 $P(B\mid A)$。
- 把不相关当成独立；协方差为零也不总能推出独立。
- 用训练集频率直接代表线上概率，忽略分布变化。
- 看到相关性就宣称因果关系。

## 练习与自测

1. 一个工具单次成功率 0.8，若两次独立，至少一次成功概率是多少？答：$1-0.2^2=0.96$。说明独立假设何时不成立。
2. 画 1000 个请求的 2×2 表：真实异常/正常 × 告警/未告警，复算贝叶斯例子。
