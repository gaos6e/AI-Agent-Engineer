# 示例客户端

## 核心结论

- 示例客户端页面用于查看哪些应用已经支持 MCP，以及它们支持的集成方式。
- 客户端能力差异很大：有的偏 IDE，有的偏桌面助手，有的偏开发调试或企业集成。
- 选择客户端时要看 transport 支持、server 配置方式、权限提示、安全策略和工具发现体验。

## 主要内容

- 页面按客户端列出 MCP 支持情况和相关链接，是评估生态兼容性的入口。
- 如果要给自己的应用添加 MCP 支持，应先读 [[05-构建 MCP 客户端|构建 MCP 客户端]] 和 [[01-客户端最佳实践|客户端最佳实践]]，再参考示例客户端。
- 对用户而言，示例客户端能帮助判断某个 MCP server 能否直接在现有工具中使用。

## 关键概念 / 流程 / 注意事项

- 客户端列表会随生态变化而更新，具体支持能力以源页面和目标应用文档为准。
- 不要只看是否支持 MCP，还要确认是否支持本地 server、远程 server、OAuth、权限确认和日志排查。
- 企业环境应优先评估审计、策略管理和 connector 管理能力。
