# 客户端最佳实践

## 核心结论

- MCP 客户端的难点不只是连上 server，而是如何在大量工具中做发现、筛选、缓存和安全执行。
- Progressive Tool Discovery 适合工具很多或上下文预算有限的客户端。
- Programmatic Tool Calling / Code Mode 可以增强复杂工作流，但必须有沙箱和权限约束。

## 主要内容

- 渐进式发现让客户端先暴露少量高价值工具或按需查询工具列表，避免把全部工具一次性塞进模型上下文。
- 动态 server 管理要求客户端能处理 server 增删、工具变化通知、连接失败和缓存失效。
- 代码模式通常让模型生成程序化调用计划，再在受控运行环境中执行，适合复杂多步工具编排。

## 关键概念 / 流程 / 注意事项

- 工具越多，越需要分层发现、命名规范和 prompt caching 策略。
- 沙箱不是可选项：文件、网络、环境变量、凭据和进程权限都要受控。
- 错误处理要给模型可恢复的信息，但不要泄露敏感配置或内部系统细节。
