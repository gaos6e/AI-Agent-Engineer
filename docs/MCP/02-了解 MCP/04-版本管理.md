# 版本管理

## 核心结论

- MCP 使用按日期标识的协议版本，客户端和服务器通过初始化阶段协商共同支持的版本。
- 版本管理的重点是互操作性：新功能应通过能力声明和协商逐步启用。
- 实现方需要明确自己支持的协议 revision，并在升级时处理兼容性。

## 主要内容

- Revision 表示协议规范的一次发布。客户端和服务器都应声明并遵守自己实现的 revision。
- Negotiation 发生在连接初始化期间，双方确认协议版本和能力集合后再进入正常交互。
- SDK 通常会隐藏一部分版本协商细节，但应用仍要关注依赖版本和部署环境中的 server/client 兼容性。

## 关键概念 / 流程 / 注意事项

- 不要假设所有 MCP 客户端都支持最新功能；应根据 capability 判断。
- 升级 server 时要保留清晰的最低客户端版本说明。
- 排查兼容问题时，先记录 client、server、SDK 和协议 revision。
