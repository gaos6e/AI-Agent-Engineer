# 调试

## 核心结论

- MCP 调试应先区分 server 启动失败、协议初始化失败、工具执行失败和客户端配置失败。
- 日志、Inspector、客户端 server status、工作目录和环境变量是最常用的证据来源。
- 安全日志要足够定位问题，但不能泄露 token、密钥、用户数据或内部路径细节。

## 主要内容

- server 侧应使用结构化 logging，记录初始化、请求、外部依赖错误和工具调用结果摘要。
- 常见问题包括工作目录不一致、环境变量缺失、server 初始化异常、连接断开和标准输出混入非协议文本。
- 在 Claude Desktop 等客户端中，还需要检查 server 状态、应用日志和必要时的开发者工具。

## 关键概念 / 流程 / 注意事项

- 先用最小命令直接启动 server，确认它能在当前环境运行，再接入 MCP host。
- stdio server 不应把普通日志写到 stdout 干扰协议流，日志应走 stderr 或文件。
- 每次修复后用 [[01-MCP Inspector|MCP Inspector]] 做最小回归，再回到目标客户端验证。
