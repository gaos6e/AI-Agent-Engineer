# 安全最佳实践

## 核心结论

- MCP 安全要围绕最小权限、用户同意、token 边界、会话隔离和工具副作用控制设计。
- 主要风险包括 confused deputy、token passthrough、SSRF、session hijacking、本地 server 被攻陷和 scope 过宽。
- 客户端和服务器都必须把能力暴露、授权、日志和错误处理当作安全边界。

## 主要内容

- Confused deputy 问题来自一个组件被诱导替攻击者使用自己的权限访问资源，缓解方式包括明确 audience、用户确认和细粒度授权。
- Token passthrough 会把客户端或用户 token 扩散到不应信任的 server，缓解方式是使用正确的 OAuth flow、限定 scope 和避免复用 bearer token。
- SSRF 与本地 server 风险要求限制 URL、网络访问、文件路径、命令执行和环境变量读取。
- Session hijacking 要通过安全 session ID、传输加密、状态绑定和对 prompt injection 的防护来降低风险。

## 关键概念 / 流程 / 注意事项

- 工具描述不能代替权限控制；模型可能被诱导调用高风险工具。
- 本地 MCP server 应像本地代理程序一样管理，不要默认给予全盘文件和 shell 权限。
- 日志中避免保存用户原文、密钥、token、完整路径和敏感响应。
