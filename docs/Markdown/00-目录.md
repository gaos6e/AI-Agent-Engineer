# Markdown

## 知识库简介

Markdown 是用纯文本表达标题、段落、列表、链接、代码和表格的轻量标记方式。AI Agent Engineer 会用它维护 README、运行手册、工具契约、提示词说明、实验记录和 Obsidian 知识库。真正目标不是背符号，而是写出：源文件本身可读、目标宿主能稳定渲染、链接可维护、步骤可执行、证据边界清楚的工程文档。

本库把 7 课正式路线与 3 份参考材料分开。[[Markdown/Markdown 教程|Markdown 完整教程]] 和 [[Markdown/Mermaid|Mermaid 完整教程]] 保留为大型查阅层；学习时先走短课程，再按问题定位章节。动态版本与大型教程的兼容提醒统一见 [[Markdown/参考资料、版本与兼容性说明|参考资料、版本与兼容性说明]]。

## 在总路线中的位置

本库位于工程基础第 6 站，上接 [[JSON/00-目录|JSON]]，下接 [[Git/00-目录|Git]]。后续提示词、RAG、Agent、评测与生产化课程都要依赖这里建立的结构、链接、证据和安全写作习惯。

## 学习目标

- 使用标题、段落、列表、引用、表格、链接、图片与嵌套代码围栏。
- 从“源文本—解析器—宿主渲染”解释换行、缩进、围栏和兼容差异。
- 区分 CommonMark 0.31.2、GFM 0.29-gfm、Obsidian 扩展与平台功能。
- 使用完整路径 wikilink、标题/块链接、embed 和相邻附件规则。
- 设计类型清楚、无重复键且不含敏感信息的 Properties。
- 用 callout 和 Mermaid 表达必要语义，不让视觉元素替代正文契约。
- 写出可供初学者执行和审查的 Agent 工程文档，区分预期、实测、建议、推断与未知。

## 前置知识

只需会使用文本编辑器和文件夹；不要求编程。建议用 Obsidian 源码模式或实时预览与阅读视图对照观察。涉及命令的练习以 Windows 11、PowerShell 7 为主，但本课程不会创建虚拟环境、安装依赖或调用外部 API。

## 掌握标准

- [ ] 能用标题层级、段落和列表组织长文，不靠粗体或空格制造结构。
- [ ] 能解释块结构与行内结构，并修复换行、缩进和嵌套围栏问题。
- [ ] 能给常用语法标注 CommonMark、GFM、Obsidian 或平台层级。
- [ ] 能创建无歧义的完整路径笔记/标题/块链接，并嵌入真实目标。
- [ ] 能写无重复键、类型稳定且不含秘密的 Properties。
- [ ] 能在当前 Obsidian 阅读视图验证 Mermaid，并记录宿主版本未知项。
- [ ] 能写一篇包含目标、契约、步骤、故障、安全、验证和来源的运行手册。
- [ ] 能让另一位初学者仅靠文档回答“从哪开始、何时停止、什么算成功”。
