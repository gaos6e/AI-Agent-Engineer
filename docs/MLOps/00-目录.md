# MLOps

## 知识库简介

MLOps（Machine Learning Operations）把机器学习模型从一次性实验变成可重复、可审查、可发布、可回滚的工程系统。它不只是“把模型部署成接口”：问题定义、数据与标签版本、特征代码、训练环境、模型制品、评测证据和线上反馈必须能沿同一条链路追溯。

本知识库聚焦经典机器学习与深度学习模型的生命周期。LLM 复合应用特有的 Prompt、检索器、工具和模型供应方编排由 [[LLMOps/00-目录|LLMOps]] 负责；日志、指标、Trace、SLO 与告警由 [[运行监控/00-目录|运行监控]] 展开。

> [!important] 版本事实与厂商建议分开看
> - **版本事实**：2026-07-14 查询 PyPI 时，MLflow 最新稳定版为 `3.14.0`，发布于 2026-06-17，声明 Python `>=3.10`。本课程不安装 MLflow，版本号只用于建立动态资料快照。
> - **厂商架构建议**：Google Cloud 的 MLOps 架构文章讨论 CI、CD、持续训练（CT）和自动化成熟度，页面标注最后审核时间为 2024-08-28。它是有价值的厂商参考，不是所有团队必须照搬的等级认证。
> - **标准状态**：NIST AI RMF 1.0 发布于 2023 年；NIST 当前页面同时说明该框架正在修订。因此本课程引用 1.0 时会写明版本，不把它表述成永久不变的最新要求。

## 在总路线中的位置

建议先掌握机器学习的训练—验证—测试流程、Git 和基本 API，再学习本知识库。MLOps 位于“模型能训练”与“模型能长期稳定提供价值”之间，也是后续 LLMOps、评测体系和 AI 治理的工程底座。

## 学习目标

完成后，你应能：

- 为一次训练记录数据、代码、参数、环境、指标和制品的可追溯证据；
- 设计数据检查、训练、评测、注册、发布与回滚管线；
- 区分模型注册、质量门、部署策略与线上监控各自解决的问题；
- 解释数据漂移、概念漂移和模型质量下降为何不能互相替代；
- 用一个离线项目实现候选模型晋级判断，并输出可审计结果。

## 前置知识

- 能运行 Python 3 脚本，理解函数、字典、JSON 与异常处理；
- 理解训练集、验证集、测试集、分类指标和过拟合；
- 会使用 Git 查看提交标识，知道虚拟环境与依赖文件的作用；
- 对 HTTP 服务、批处理任务和容器有概念即可，不要求已有 Kubernetes 经验。

## 环境路线：先 venv + pip，再选平台

不要在 Obsidian vault 内创建 `.venv`。若要尝试 MLflow，请先在独立练习目录建立环境并锁定快照版本：

```powershell
$project = Join-Path $HOME "Projects\mlops-learning"
New-Item -ItemType Directory -Path $project -Force | Out-Null
Set-Location $project

py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install "mlflow==3.14.0"
python -c "from importlib.metadata import version; print(version('mlflow'))"
python -m pip check
```

此安装命令未在本知识库离线项目中执行，也不需要云凭据。未来使用时应重新核对 PyPI、发行说明、Python 约束和组织安全策略；理解 `venv + pip` 后，再按团队约定使用 `uv`、容器或锁文件。

平台是实现手段，不是学习顺序：MLflow 可用于 Tracking 与 Registry；Kubeflow Pipelines 可在 Kubernetes 上组织组件、参数、制品与控制流；KServe 提供 Kubernetes 模型服务抽象；Kubernetes Deployment 管理工作负载发布。初学阶段先用普通 Python 把契约和判定写清，再决定是否需要这些平台。

## 推荐学习顺序

1. [[MLOps/01-基础与生命周期/01-实验与可复现追踪|实验与可复现追踪]]：先学会回答“这个结果从哪里来”。
2. [[MLOps/01-基础与生命周期/02-数据、特征与训练管线|数据、特征与训练管线]]：把数据契约和训练步骤变成可重复流程。
3. [[MLOps/01-基础与生命周期/03-模型注册与候选晋级|模型注册与候选晋级]]：理解版本、别名、签名与晋级证据。
4. [[MLOps/02-生产工程/04-CI、CD与自动化质量门|CI、CD 与自动化质量门]]：把测试和评测放进发布路径。
5. [[MLOps/02-生产工程/05-部署、Canary与回滚|部署、Canary 与回滚]]：选择批处理、在线、影子或渐进发布方式。
6. [[MLOps/02-生产工程/06-模型监控、漂移与反馈|模型监控、漂移与反馈]]：观察数据、预测和有标签质量的变化。
7. [[MLOps/02-生产工程/07-故障、回滚与再训练决策|故障、回滚与再训练决策]]：用证据决定止损、修复还是再训练。
8. [[MLOps/02-生产工程/08-平台安全与治理|平台安全与治理]]：把身份、秘密、供应链、审计和风险责任放入生命周期。
9. [[MLOps/03-项目与自测/08-离线模型晋级项目与自测|离线模型晋级项目与自测]]：完成血缘、候选晋级、漂移调查与回滚判定。

## 动手实践与项目入口

主项目见 [[MLOps/03-项目与自测/08-离线模型晋级项目与自测|离线模型晋级项目与自测]]。项目只使用 Python 标准库，读取候选制品 manifest 和线上观察窗口，对数据、代码、环境与训练配置血缘、测试、指标、签名、漂移和回滚条件进行严格检查，并输出带策略版本与证据指纹的理由。

## 掌握标准

- [ ] 能从一个预测结果反查数据快照、代码提交、环境、参数和模型制品；
- [ ] 能说明“可复现”不等于所有硬件上逐位相同；
- [ ] 能为数据、模型输入输出和发布制品写出最小契约；
- [ ] 能区分 CI、持续训练（CT）和 CD 的职责；
- [ ] 能根据业务风险设计候选晋级、Canary 观察和回滚证据，而非照抄固定阈值；
- [ ] 能区分数据漂移、概念漂移与真实性能下降；
- [ ] 能为训练平台和模型服务定义身份、秘密、制品完整性、审计和人工审批边界；
- [ ] 能运行项目中的通过与阻断样例，并读懂每条判断理由。
