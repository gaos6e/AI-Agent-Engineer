# 厂商 AI API 参考目录

## 这组笔记是什么

这里保存 10 家厂商的 Python 调用参考，用于比较 endpoint、SDK、模型标识、流式事件、结构化输出和工具调用差异。它们是**动态参考层**，不替代 [[API/00-目录|API 通用主线]]，也不应成为零基础学习的第一站。

个别模型名、别名、SDK 方法、beta 标记、计费和区域 endpoint 会变化。各笔记保留自己的官方来源与 `source_checked`；本轮于 **2026-07-22** 重新核对了目录和除 OpenAI 外各厂商的主入口、默认教学模型或关键能力边界。OpenAI 本轮只局部复核了工具指南，其页面仍以自身 `source_checked: 2026-07-20` 为准。特别是 Google：Interactions API 已于 2026 年 6 月 GA 且是新项目推荐入口，原 `generateContent` API 仍 fully supported 但属于 legacy；对应笔记已把两条 API family 分层，不能把示例字段交叉套用。真正运行前仍要重新查看目标厂商文档与账户可用模型。

## 验证边界

- 已做：官方来源核对、Python fenced code 语法解析、内部链接解析、凭据示例安全审查。
- 未做：安装全部厂商 SDK、创建十家账户、使用真实 key、发出可能计费的请求、比较实时价格或性能。
- 因此“来源已核对”不等于“网络调用已执行”。只有本机 `API/examples/` 项目完成了真实 loopback HTTP 运行。

## 跨厂商最低运行合同

同为「OpenAI 兼容」通常只说明某个请求/响应外形可以复用，**不**保证模型 ID、流式事件、工具回传、结构化输出、数据保留、重试、计费或权限语义相同。每次真实调用至少把下列信息作为一次可追溯的 provider contract，而不要只保存最终文本：

| 记录项 | 作用 | 不能替代什么 |
| --- | --- | --- |
| 官方文档 URL、`source_checked`、SDK 版本 | 说明本次代码依据的文档快照 | 对账户/区域实际可用性的证明 |
| provider、endpoint/region、精确 `model` | 固定请求落点与模型语义 | 业务版本、质量基线或数据驻留合规 |
| 流终态、拒答/工具项、结构化字段校验 | 防止把 delta、文本片段或 JSON 语法当完成结果 | 事实核验、身份授权或副作用批准 |
| provider `request_id` 与本地 `operation_id` | 分别关联供应商诊断和本地业务意图 | 彼此、用户身份、幂等/去重 ledger 或最终 outcome |
| tool result、引用、用量与错误类别的脱敏摘要 | 支持评测、排障和成本归因 | 完整 prompt、凭据、原始敏感输入或事实证明 |

工具、结构化输出或远程搜索只是模型能力。进入写库、发消息、支付、部署等副作用前，仍要按 [[Tool Calling（含 Function Calling）/00-目录|Tool Calling]]、[[JSON/00-目录|JSON]] 和 [[LLM API集成/00-目录|LLM API 集成]] 的本地 schema、可信 actor、对象级 ACL、审批、业务幂等和 receipt/outcome 核验执行。

## 凭据只在当前会话安全输入

不要把真实 key 写进 Markdown、`.py`、命令历史、截图或 Git。PowerShell 7 可在当前进程用遮罩提示读取：

```powershell
$env:EXAMPLE_API_KEY = Read-Host 'EXAMPLE_API_KEY' -MaskInput
try {
    python .\your_example.py
}
finally {
    Remove-Item Env:EXAMPLE_API_KEY -ErrorAction SilentlyContinue
}
```

把 `EXAMPLE_API_KEY` 换成目标厂商的变量名。这里不使用 `setx`：它写入用户环境、只对后续新进程生效，并容易诱导学习者把真实值留在命令历史。团队和生产部署应使用平台 secret manager；若项目使用 `.env`，只提交空值的 `.env.example`，并确认真实 `.env` 从未被 Git 跟踪。

## 使用检查清单

选择一家供应商时，先完成：

- [ ] 在官方文档确认 base URL、地域、SDK 包名和当前模型 ID。
- [ ] 确认输入、输出、流式事件与工具调用 schema，而非套用另一家格式。
- [ ] 设置 connect/read timeout、整体 deadline 和明确重试层。
- [ ] 记录 source/SDK/模型核对日期，并区分文档核对与实际运行。
- [ ] 使用测试项目或低风险非敏感输入，确认错误类型和用量字段。
- [ ] 不把 key、完整 prompt、文件或响应正文写入普通日志。
