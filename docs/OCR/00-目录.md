# OCR

## 知识库简介

OCR（Optical Character Recognition，光学字符识别）把照片、扫描件或 PDF 页面中的可见文字转换为可搜索、可计算的数据。真正可用的系统不只是“图片转字符串”：它还要保留页码、文本框、阅读顺序、表格结构、置信度和来源，以便 Agent 能引用原文、触发人工复核并追踪错误。

本知识库从成像与版面直觉开始，逐步覆盖识别、后处理、评测、隐私部署，最后用一个完全离线的结构化结果审计器串起全流程。课程中的产品能力可能随版本变化；动态资料已于 **2026-07-22** 核验，使用前仍应查目标版本的官方文档。

## 在总路线中的位置

OCR 位于“扩展应用与复杂协作”阶段。它把图像中的文档变成 [[文档解析/00-目录|文档解析]] 和 [[RAG/00-目录|RAG]] 可消费的结构化输入，也为 [[多模态AI/00-目录|多模态 AI]] 提供可审计的文字通道。

## 学习目标

- 解释从成像、预处理、版面分析到文字识别和后处理的完整流水线。
- 为文本块、表格、阅读顺序、页码、置信度设计稳定的结果契约。
- 使用 CER、WER、字段与结构指标定位错误，而不是只看一个总分。
- 设计低置信度人工复核、隐私边界和可回归的离线测试集。
- 完成一个能校验结构、顺序、文本误差与复核队列的标准库项目。

## 前置知识

- 建议先学 [[Python基础/00-目录|Python 基础]]、[[JSON/00-目录|JSON]] 和 [[数据标注/00-目录|数据标注]]。
- 不要求图像处理或深度学习基础；相关概念会先给直觉，再说明工程边界。

## 推荐学习顺序

1. [[OCR/01-基础与数据/01-OCR全流程与输入契约|OCR 全流程与输入契约]]：先确定输入、输出和可追溯字段。
2. [[OCR/01-基础与数据/02-成像原理与预处理|成像原理与预处理]]：理解分辨率、倾斜、噪声和二值化为何影响识别。
3. [[OCR/01-基础与数据/03-文字检测识别与数据标注|文字检测、识别与数据标注]]：区分“文字在哪里”和“写了什么”。
4. [[OCR/02-工程与质量/04-版面表格与阅读顺序|版面、表格与阅读顺序]]：避免多栏、表格和页眉被拼错。
5. [[OCR/02-工程与质量/05-语言后处理与置信度|语言后处理与置信度]]：做有边界的规范化和复核分流。
6. [[OCR/02-工程与质量/06-评测错误分析与人工复核|评测、错误分析与人工复核]]：建立文本、字段和结构三层评测。
7. [[OCR/02-工程与质量/07-隐私部署与运行排查|隐私部署与运行排查]]：把数据最小化、日志和回滚纳入设计。
8. [[OCR/03-项目与自测/08-项目-结构化OCR结果审计|项目：结构化 OCR 结果审计]]：运行离线项目并完成验收。

## 掌握标准

- [ ] 能画出并解释 OCR 的检测、识别、版面、后处理和复核数据流。
- [ ] 能说明单纯提高图像尺寸为何不一定提升质量，并设计可回滚的预处理对照。
- [ ] 能为多栏页面和表格定义阅读顺序与结构输出。
- [ ] 能手算一个 CER/WER 示例，并解释为什么分母为参考文本长度。
- [ ] 能把低置信度、关键字段规则和抽样复核组合为人工队列。
- [ ] 能运行项目、解释报告，并为一种新文档类型补一个匿名夹具。
