# 离线协作模拟项目

## 项目目标

用一个完全确定性的调度器模拟多 Agent 协作，而不调用模型。你将验证：依赖满足后才运行、角色权限受限、失败只有限重试、预算是全局的、重复消息不重复提交、每一步都有追踪事件。

## 文件

- [collaboration_simulator.py](%E5%A4%9AAgent%E5%8D%8F%E4%BD%9C/03-%E9%A1%B9%E7%9B%AE%E4%B8%8E%E8%87%AA%E6%B5%8B/examples/collaboration_simulator.py)：标准库实现，读取场景并运行状态机。
- [scenario.json](%E5%A4%9AAgent%E5%8D%8F%E4%BD%9C/03-%E9%A1%B9%E7%9B%AE%E4%B8%8E%E8%87%AA%E6%B5%8B/examples/scenario.json)：合成角色、任务、失败计划与预算。
- [test_collaboration_simulator.py](%E5%A4%9AAgent%E5%8D%8F%E4%BD%9C/03-%E9%A1%B9%E7%9B%AE%E4%B8%8E%E8%87%AA%E6%B5%8B/examples/test_collaboration_simulator.py)：覆盖成功、权限拒绝、重试与预算停止。

这些夹具不包含真实会话、凭据或外部数据。

## 场景结构

角色声明 capabilities；任务声明 owner、requires、capability、max_attempts 和 outcome_plan。outcome_plan 是确定性结果序列，例如 transient_error 后 success，用来测试重试，不代表模型推理。

调度器每轮：

1. 找出依赖已成功的 pending 任务；
2. 检查全局 step_budget；
3. 检查任务所有者是否具有所需 capability；
4. 产生一次预设结果；
5. 根据结果和 max_attempts 进入 succeeded、pending、failed 或 denied；
6. 若没有任务可推进，区分成功、上游失败阻塞和依赖死锁。

## 运行

在 PowerShell 7 中进入本目录后执行：

~~~powershell
python -B .\examples\collaboration_simulator.py .\examples\scenario.json
python -B -m unittest discover -s .\examples -p 'test_*.py' -v
python -B -O -m unittest discover -s .\examples -p 'test_*.py'
python -B -W error -m unittest discover -s .\examples -p 'test_*.py'
~~~

也可把报告写入临时路径：

~~~powershell
python -B .\examples\collaboration_simulator.py .\examples\scenario.json --output "$env:TEMP\multi-agent-report.json"
~~~

脚本不访问网络，也不在知识库内生成运行产物。

成功返回退出码 0；工作流以 `failed`、`deadlock` 或 `budget_exhausted` 等非成功终态结束时返回 1；场景契约错误返回 2。56 项测试覆盖严格 JSON、角色/任务合同、权限、重试、死锁、幂等、预算、事件序列和 CLI。

## 读懂输出

报告包含 status、steps_used、tasks 和 events。检查：

- transient_error 之后是否只有允许次数内的下一次 attempt；
- 缺少 capability 是否进入 denied，而不是换角色绕过；
- 依赖失败是否让下游 blocked；
- event 是否带 task_id、attempt、from、to 和 reason；
- steps_used 是否不超过 step_budget。

## 改造练习

1. 在场景中新增一个和汇总任务并行的审查任务，观察关键路径。
2. 把某角色 capability 删除，确认策略拒绝。
3. 把 step_budget 调小，确认终态为 budget_exhausted。
4. 制造 A 依赖 B、B 依赖 A，确认报告指出 deadlock。
5. 增加 idempotency_key 去重测试：重复结果不得多次改变状态。

## 自测题

1. 如果任务已经 succeeded，迟到的 failed 结果应如何处理？
2. 为什么全局预算不能由各 Agent 自己统计？
3. 一个下游任务因依赖 failed 无法运行，它自身应记 failed 还是 blocked？
4. 怎样把此模拟器改为可恢复而不重复副作用？

## 局限

项目没有并发线程、网络传输、真实模型、持久队列或人工审批 UI。它验证的是控制语义。把它迁移到框架前，应保留相同状态、预算和验收测试。
