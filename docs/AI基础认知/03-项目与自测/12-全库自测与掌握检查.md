# 全库自测与掌握检查

## 使用方法

先关闭课程笔记，用自己的例子回答。每题至少包含“定义/判断—具体例子—工程后果”三部分；只复述名词不算通过。回答后按题目链接回到相应条目核对，并记录仍然混淆的概念。

建议评分：

- `0`：无法回答或关键判断错误；
- `1`：定义大致正确，但没有例子或工程后果；
- `2`：定义、例子和边界完整，能说明何时不适用。

总分 32 分以上，且标为“关键”的题不得为 0，才进入后续主线。分数只用于学习诊断，不是能力认证。

3. 规则自动化、单次 LLM、固定工作流和 Agent 怎样区分？

6. 训练目标、任务指标和系统结果可能怎样错位？请给一个 JSON 格式正确但业务错误的例子。

## 五、方案、上线与责任

20. **关键**：面对“让 AI 自动筛选最优秀候选人”，你会先问哪些问题，如何缩小范围？
21. 自主性阶梯为什么应从只读信息或草稿开始，而不是直接自动执行？
22. 价值门、质量门和风险门分别可能阻止什么发布？
23. 影子模式、降级、回滚和停用有什么区别？
24. 第三方模型或工具服务发生无报错更新时，为什么仍要重新评测？
25. **关键**：人工审核在什么条件下才是有效控制？
26. 可信 AI 为什么不能压缩成一个总分？隐私、透明、可解释与公平可能怎样权衡？
27. 高影响场景为什么需要专业复核、证据访问和申诉路径？

核对：[[AI基础认知/02-工程决策/08-何时使用AI与系统形态选择|何时使用 AI 与系统形态选择]]、[[AI基础认知/02-工程决策/09-从原型到上线与退出|从原型到上线与退出]]、[[AI基础认知/02-工程决策/10-负责任使用与风险控制|负责任使用与风险控制]]。

## 六、综合迁移题

28. 为“内部政策问答”画出输入、权限、检索、模型、引用校验、人工和日志，并标出信任边界。
29. 为上述系统写一个规则/搜索基线、六个测试切片、一个质量门和一个停用条件。
30. 同一需求下，什么时候用固定工作流，什么时候才考虑 Agent？
31. 如果模型性能提高但用户完成率下降，你会按什么顺序调查？
32. 如果无法可靠测量某项高后果风险，团队应该怎样记录并决定，而不是默认低风险？

这些题没有唯一技术答案，但必须有明确目标、证据、责任和安全退出。若回答只包含“优化提示”“换更强模型”或“让人工检查”，返回相应课程重做练习。

## 掌握检查

- [ ] 能用自己的例子解释 AI、ML、DL、LLM、工作流和 Agent。
- [ ] 能区分数据、参数、训练、推理、上下文与反馈。
- [ ] 能设计没有明显泄漏的开发/测试边界和关键切片。
- [ ] 能把失败归因到系统层，而不是笼统归因于“模型不够强”。
- [ ] 能比较规则、ML、LLM、工作流与 Agent，并选择最小可用形态。
- [ ] 能定义 baseline、离线门槛、试运行、监控、回滚和停用。
- [ ] 能说明人工监督何时有效，责任属于谁。
- [ ] 能为输出契约、证据、权限和外部动作设计确定性校验。
- [ ] 已完成 [[AI基础认知/03-项目与自测/11-综合项目-会议行动项助理|综合项目：会议行动项助理]]，并达到项目量表要求。
