# Linux 常用指令遗留速查

> [!warning] 模板不是可直接运行的脚本
> 本页保留原有速查内容，尖括号表示“需要替换的占位符”。完整解释、安全护栏与可运行练习见 [[Linux命令/00-目录|Linux命令统一入口]]。不要把占位符原样粘贴，也不要对 `/`、家目录、挂载点或真实项目执行递归删除。

## tmux

### 新建会话

```text
tmux new -s <session_name>
```

### 进入会话

```text
tmux attach -t <session_name>
```

### 查看已有会话

```bash
tmux ls
```

### 删除某个会话

```text
tmux kill-session -t <session_name>
```

## 文件夹操作

### 删除文件夹及其内容

```text
rm -r <folder_path>
```

### 强制递归删除签名（仅供识别）

```text
rm -rf <folder_path>
```

强制递归删除会跳过一部分确认，变量为空或路径算错时可能造成不可逆损失。本知识库不提供可直接运行的版本；先学习 [[Linux命令/03 文件和目录操作|文件和目录操作]] 中的绝对路径、所有权、符号链接与目标边界检查。

## 压缩/解压文件夹

### 压缩为 `.tar`

```text
tar -cvf <archive_name>.tar <folder_path>
```

### 压缩为 `.tar.gz`

```text
tar -czvf <archive_name>.tar.gz <folder_path>
```

### 压缩为 `.zip`

```text
zip -r <archive_name>.zip <folder_path>
```

### 解压 `.tar`

```text
tar -xvf <archive_name>.tar
```

### 解压 `.tar.gz`

```text
tar -xzvf <archive_name>.tar.gz
```

### 解压 `.zip`

```text
unzip <archive_name>.zip
```

解压不可信归档前先查看清单，并把目标限制在新建的空目录中；详见 [[Linux命令/13 压缩与解压|压缩与解压]]。

## Conda 环境管理

### 查看 Conda 版本

```bash
conda --version
```

### 查看所有环境

```bash
conda env list
```

当前激活的环境会以 `*` 标记。

### 创建环境

```text
conda create --name <env_name> python=<python_version>
```

例如，创建使用 Python 3.11 的 `myenv` 环境：

```bash
conda create --name myenv python=3.11
```

### 激活和退出环境

```text
conda activate <env_name>
conda deactivate
```

### 查看环境中的软件包

```text
conda list --name <env_name>
```

### 搜索、安装、更新和卸载软件包

```text
conda search <package_name>
conda install --name <env_name> <package_name>
conda update --name <env_name> <package_name>
conda remove --name <env_name> <package_name>
```

从指定 channel 安装软件包：

```text
conda install --name <env_name> --channel conda-forge <package_name>
```

### 克隆环境

```text
conda create --name <new_env_name> --clone <source_env_name>
```

### 导出和复现环境

只导出手动安装的主要依赖，通常更适合跨平台复现：

```text
conda env export --name <env_name> --from-history > environment.yml
```

根据环境文件创建或更新环境：

```bash
conda env create --file environment.yml
conda env update --file environment.yml --prune
```

### 删除整个环境

```text
conda remove --name <env_name> --all
```

删除前先用 `conda env list` 核对环境名称。需要使用 `pip` 时，先激活目标 Conda 环境，再执行 `python -m pip ...`，避免把软件包安装到错误的 Python 环境中。
