# 04 查看文件内容

## 学习目标

- 能直接查看文本文件内容。
- 能分页查看大文件。
- 能查看文件开头、结尾和实时追加内容。

## 命令总览

| 命令 | 用途 | 含义 | 基本用法 | 例子 |
|---|---|---|---|---|
| `cat` | 输出整个文件 | concatenate，常用于直接打印小文件 | `cat 文件` | `cat README.md` |
| `less` | 分页查看文件 | 适合浏览大文件，可上下翻页和搜索 | `less 文件` | `less app.log` |
| `head` | 查看文件开头 | 默认显示前 10 行 | `head [选项] 文件` | `head -n 20 app.log` |
| `tail` | 查看文件结尾 | 默认显示后 10 行，可实时跟踪 | `tail [选项] 文件` | `tail -f app.log` |
| `wc` | 统计换行、单词、字节或字符 | 默认显示换行数、单词数、字节数；`-m` 才是字符数 | `wc [选项] 文件` | `wc -l app.log` |

## 常用场景

查看小文件：

```bash
cat config.txt
```

分页查看大日志：

```bash
less app.log
```

查看开头和结尾：

```bash
head -n 20 app.log
tail -n 50 app.log
```

实时看日志新增内容：

```bash
tail -F -- app.log
```

统计文件行数：

```bash
wc -l app.log
```

## 新手注意

- `cat` 不适合直接打开很大的日志文件，容易刷屏。
- `less` 里按 `/关键词` 搜索，按 `n` 跳到下一个匹配，按 `q` 退出。
- `tail -f` 会持续占用终端，按 `Ctrl-c` 停止。
- GNU `tail -F` 会按名称重试并处理常见日志轮转；并非所有平台选项一致。
- `wc -l` 数换行符；末尾没有换行符的最后一段文本不会增加该计数。`wc -c` 数字节，`wc -m` 数字符。
- 日志可能含用户输入、token、内部地址与命令行，不要把整份文件直接公开。

## 日志查看练习

在综合项目的虚构日志上比较 `head -n 2`、`tail -n 2`、`wc -l` 和 `less`。说明每条命令读取的范围、是否持续运行、输出是否适合进入报告。
