# Linux命令

## 知识库简介

Linux 命令行是部署、容器、服务器排障和远程运行 Agent 服务的基础。本库保留既有速查层，并补齐 Shell 心智模型、服务与日志、网络与端口、软件包管理以及综合实践。课程先教“确认环境、身份、当前目录和对象，再执行命令”的安全习惯，再进入文件、文本、管道、权限、进程、资源、服务、网络、环境变量与 tmux。

主要本机环境仍是 Windows 11 与 PowerShell 7。Linux 示例应在 WSL、容器或明确的远程 Linux shell 中运行；不要把 Bash 与 PowerShell 语法混在同一个命令上下文。

> [!important] 四层兼容边界
> 本库区分 POSIX 可移植语法、GNU 工具扩展、Linux 专属工具和 Bash builtin。`systemd`、`procps-ng`、`util-linux`、`iproute2` 并非所有 Unix 或最小容器都有；macOS/BSD 的选项也可能不同。先确认环境与本机帮助，再执行示例。

## 在总路线中的位置

它位于工程基础阶段。日常本机文件操作优先使用 PowerShell；当任务进入 Linux 服务器、Docker 容器或 WSL 时，再使用本库。后续 MLOps、LLMOps、运行监控和自动化都会依赖这些能力。

## 学习目标

- 识别当前 shell、用户、主机、目录和命令来源。
- 安全完成路径、文件、查看、查找、文本处理与管道任务。
- 理解标准输入、标准输出、标准错误和退出码。
- 读取权限、进程、磁盘、内存与负载信息，不靠猜测排障。
- 管理会话、环境变量和长任务，同时避免泄露凭据。
- 读取服务、日志、监听端口、DNS、TCP、TLS 与 HTTP 层证据。
- 识别发行版与软件包来源，避免盲目下载后直接执行。
- 为本地 loopback Agent 模拟服务完成一次受控排障演练并形成证据报告。

## 前置知识

会使用 PowerShell 切换目录并理解文件路径即可；建议先完成 [[Git/00-目录|Git]]。不要求已有服务器经验。所有练习应在自有 WSL、容器或明确授权的 Linux 环境进行。

### 环境确认

从 PowerShell 进入 WSL 前先检查：

```powershell
wsl --status
wsl --list --verbose
```

进入 Linux shell 后：

```bash
whoami
hostname
pwd
printf 'bash_version=%s\n' "${BASH_VERSION:-not-bash}"
ps -p "$$" -o comm=
```

`SHELL` 常表示登录环境配置的首选 shell，不能可靠证明当前正在运行的 shell。`wsl` 的具体安装与参数会变化，实施时查 Microsoft 当前文档。远程服务器还要遵守组织访问、审计与变更流程。

## 动手实践与项目入口

- 每节只在你拥有的实验目录执行文件修改命令。
- 删除、递归、权限和进程信号命令先打印解析后的目标，并用只读命令确认。
- 综合项目见 [[Linux命令/17-Agent服务排障实践与自测|Agent 服务排障实践与自测]]，核心路径不要求管理员权限。

## 掌握标准

- [ ] 能说明命令运行于 PowerShell、WSL、容器还是远程 Linux。
- [ ] 对含空格、通配符或以 `-` 开头的路径能正确引用并使用 `--`。
- [ ] 能组合 `find`/`grep`/`sort`/`uniq`，并解释管道每一段。
- [ ] 能区分 stdout、stderr 与退出码，知道管道失败传播的注意点。
- [ ] 能只读调查日志、进程、端口相关线索、磁盘和内存，再提出行动。
- [ ] 不把 `sudo`、递归删除、宽松权限或强制终止当默认修复。
- [ ] 能安全使用环境变量和 tmux，不把秘密写进历史或命令输出。
