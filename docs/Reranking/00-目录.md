# Reranking

## 知识库简介

Reranking（重排序）接收第一阶段已授权候选，用更昂贵、能联合观察 query—document 的模型或可审计规则重新排序，再把少量高价值证据交给下游。它擅长区分“同主题但没回答”“数字或条件冲突”“精确证据与泛泛相关”，但有一个不可突破的上限：不在 candidate window 中的文档无法被重排器创造出来。

一条可上线的重排管线不只包含模型，还要包含：

- candidate window 与输出 top-n 的独立预算；
- query、候选文本、first-stage rank/score 与 source revision 契约；
- Cross-Encoder、规则、LTR 或 LLM 的选择与组合；
- 长文截断、chunk 聚合和 canonical-document 去重；
- pointwise/pairwise/listwise 数据与 hard negatives；
- 模型输出 schema、ID 完整性、非有限分数与重复结果校验；
- timeout、5xx、空响应、畸形响应和容量降级；
- candidate recall、nDCG/MRR、p95/p99、成本与 fallback 质量。

> [!important] 安全边界
> Tenant、ACL、状态、有效期与删除等硬约束应在 [[语义搜索/00-目录|语义搜索]] 阶段执行。Reranker 适合做输入防御复核，但不能让模型决定“越权内容是否相关”。正常与 fallback 都只能对同一安全候选集合排序。

## 在总路线中的位置

本库是 RAG 阶段第 28 个学习点。前一站 [[语义搜索/00-目录|语义搜索]] 负责高召回候选；本库改善候选前排的 relevance、纯度和多样性；下一站 [[RAG/00-目录|RAG]] 将重排结果压缩、引用并生成回答。[[向量数据库/00-目录|向量数据库]] 的 ANN 参数和本库模型质量必须分别评测。

## 学习目标

完成本库后，你应能：

- 计算 candidate Recall@window，并解释 reranker 的可达质量上限；
- 区分召回 window、rerank window、输出 top-n 和 RAG evidence budget；
- 解释 bi-encoder 与 cross-encoder 的交互、缓存和计算成本差异；
- 为 query/title/body/metadata 定义输入模板、tokenizer 与截断策略；
- 正确解释 logit、回归分、概率与跨 query 阈值；
- 比较规则、Cross-Encoder、LTR、pointwise/pairwise/listwise LLM；
- 从真实 first-stage 高排误召回中构造经确认的 hard negatives；
- 对长文做 passage 化或分块重排，并评测聚合与重复来源；
- 验证模型输出 ID 完整、唯一、有限且仅来自输入窗口；
- 为 timeout、错误、空/畸形输出设计确定性、安全、可观测 fallback；
- 同时报告 relevance、尾延迟、吞吐、成本、稳定性和端到端收益。

## 核心术语

| 术语 | 初学者解释 | 关键边界 |
| --- | --- | --- |
| candidate window | 实际送入 reranker 的候选集合 | 决定相关文档能否被看见 |
| candidate recall | 窗口包含的正 qrels 比例 | 是排序质量上限，不等于最终 nDCG |
| cross-encoder | query 与一篇候选联合编码打分 | 每个 pair 都要实时计算 |
| pointwise | 每篇独立打分 | 易批处理，列表比较较弱 |
| pairwise | 比较同 query 下 A 与 B | 组合数和调用成本增加 |
| listwise | 一次判断候选列表 | 受窗口、顺序、格式和位置偏差影响 |
| hard negative | first-stage 高排但经确认不相关的候选 | 未标记候选不自动是负例 |
| output contract | 模型必须返回的 ID、分数、理由和版本结构 | 空/重复/未知/非有限都应触发 fallback |
| fallback | 模型不可用时的已评测安全路径 | 通常保留安全 first-stage 顺序 |

## 掌握标准

- [ ] 先报告 candidate Recall@window，再评价 reranker。
- [ ] First-stage、model 与 final rank/score 分开保存。
- [ ] Query/title/body 模板、tokenizer、最大长度和截断可回放。
- [ ] Cross-Encoder/LLM 分数不被默认当通用相关概率。
- [ ] Hard negatives 来自真实候选并经标注确认。
- [ ] 长文与重复来源策略有 evidence-coverage 实验。
- [ ] 输出只包含输入窗口的唯一 ID，分数有限且数量完整。
- [ ] Timeout、错误、空/畸形输出都走相同安全 fallback。
- [ ] Fallback 自身有 relevance、安全、延迟和容量测试。
- [ ] 上线门禁同时覆盖关键切片、p99、成本和端到端回答。
