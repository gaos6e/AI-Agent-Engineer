# 视频生成

## 知识库简介

视频生成（video generation）让模型依据文本、图像或既有片段产生一段随时间变化的视听内容。它不是“连续生成很多张图”这么简单：镜头、帧率、运动、身份连续、声音、字幕、后期与失败恢复共同决定交付质量。本知识库以短镜头工作流为主，强调先分镜、逐镜头生成、离线验证任务包，再由人工剪辑与复核完成作品。输入、镜头变换和发布候选还必须分别追踪版本、对象级访问控制和撤权/删除传播。

## 在总路线中的位置

它位于“扩展应用与复杂协作”的末端，综合运用提示词、多模态输入、异步任务、评测、安全和生产化知识。学习完成后，你应能让 Agent 编排视频任务，但不会把创作与发布决定完全交给 Agent。

## 学习目标

- 理解帧、帧率、时长、分辨率、镜头与时间线的关系。
- 用直觉解释视频生成中的空间质量与时间一致性。
- 把创意拆成 storyboard、shot list、生成任务、后期、音频和字幕接口。
- 设计运动、身份、连续性、音画和安全的分层验收。
- 用离线验证器检查时间线、权利信息、失败恢复、人工审核门禁，以及 `source_revision`、`transform_id`、`release_id` 的来源链。

## 前置知识

开始前建议具备以下能力：理解 [[图像生成/00-目录|图像生成]] 的构图、条件控制与质量边界；能用 [[提示词工程/00-目录|提示词工程]] 描述镜头约束；理解 [[工作流自动化/00-目录|工作流自动化]] 的异步任务与失败恢复；能用 [[JSON/00-目录|JSON]] 表达任务包。它们是按任务补齐的能力清单，不要求先完整学完四门课程。不要求剪辑软件经验；课程只讲可迁移的工作流与接口。

## 掌握标准

- [ ] 能把一个创意拆成有明确起止时间、镜头语言和连续性锚点的 shot list。
- [ ] 能解释帧内质量与跨帧一致性的区别。
- [ ] 能设计生成、下载、转码、剪辑、字幕、配音和审查之间的接口。
- [ ] 能为失败镜头局部重试，而不是每次重做整片。
- [ ] 能记录素材授权、人物同意、生成历史和内容凭证计划。
- [ ] 能在预览、评分或模型读取前落实对象级授权/ACL，并在撤权或删除时让镜头候选、成片派生物、缓存和链接按计划失效。
- [ ] 能区分合成内容披露与事实性主张；后者必须有独立证据达到 `evidence_supported`。
- [ ] 离线任务包验证器及其测试可以通过。
