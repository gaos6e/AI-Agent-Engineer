# 质量、运动与身份一致性评测

## 本节目标

用逐帧、逐镜头与成片三级评审发现不同尺度的问题，并知道自动指标的局限。

## 三级评审

### 逐帧抽查

在开头、中间、结尾和快速运动/遮挡处抽帧，检查形状、肢体、文字、Logo、背景和敏感内容。单帧合格不能证明视频合格，但能快速定位视觉崩坏。

### 镜头内播放

以正常速度、慢速和循环播放检查：运动是否连续、速度是否合理、接触关系是否成立、纹理是否闪烁、身份是否漂移、相机是否按要求运动、音画是否同步。记录发生时间，不只写“有点怪”。

### 成片与跨镜头

检查叙事、方向、道具状态、色彩、音量、字幕、转场、节奏和权利披露。跨镜头一致性来自项目状态与剪辑决策，不能只归因于单个生成模型。

## 评分维度与硬门槛

可以分别评分：提示遵循、单帧质量、运动自然度、主体身份、背景稳定、物理/因果、镜头语言、连续性、音画、可访问性、安全与权利。安全与权利维度还应核对输入 `source_revision`、对象级授权/ACL、撤权/删除传播状态，以及事实性外部陈述是否有独立证据达到 `evidence_supported`。人物同意缺失、禁用内容、ACL 不允许读取、事实性误导或证据不足、字幕关键数字错误等设为硬失败，不被平均分抵消。

## 自动指标的角色

FVD 将视频特征分布用于集合级比较，原论文通过人类研究检验了相关性；它仍依赖特征网络、样本和数据域，不是单段业务验收器。逐帧感知距离可辅助检测闪烁或编辑保持，但会受相机运动影响。光流或跟踪信号可提示运动异常，却无法判断动作是否符合创意和物理常识。

自动信号适合筛查和回归监控，最终评审要结合人类。任何自动抽帧、OCR、身份相似度或评审模型都只能在对象级授权/ACL 允许后读取素材。评估候选或模型版本时随机顺序、隐藏系统名、记录 `source_revision`、`transform_id`、失败标签和尝试次数；人工批准后才关联 `release_id`。只展示成功样本会严重高估稳定性。

## 身份与连续性检查表

为主体选择可观察锚点：面部/外形、服装、颜色、道具、左右关系和尺寸。每次遮挡恢复、转身和切镜后重新检查。真实人物还要检查是否获得生成和发布同意；“看起来不像本人”不等于没有隐私或声誉风险。

## 常见错误与排查

- 只看一遍正常速度：慢放、循环和关键帧抽查会发现不同缺陷。
- 只报平均分：保留时间点、镜头 ID 和失败类别。
- FVD 下降就宣布业务更好：确认数据、实现、样本量和人工偏好。
- 评审者看见模型名：采用匿名成对比较降低偏差。
- 评分通过就继续发布：评分不是授权；撤权/删除后应停止关联候选和评审产物的访问。
- 合成视频看似合理就作事实主张：将事实依据与视频本身分离，只有独立证据足够时才声明 `evidence_supported`。
