# 离线发布门项目与自测

## 项目目标

运行一个不调用真实 LLM API 的 LLMOps 门禁。它完成三件事：

1. 严格校验复合发布清单，拒绝缺字段、未知字段、错误类型和常见浮动别名；
2. 把离线测试、任务质量、安全样本、延迟、估算成本、Trace、脱敏、Canary、回滚和人工审批转成 `promote/block`；
3. 把模拟线上观察转成 `continue/investigate/pause/fallback/rollback/human_review`，并为参与决策的证据生成稳定指纹。

项目文件：

- [[LLMOps/03-项目与自测/examples/release_gate.py|release_gate.py]]：严格契约、决策逻辑和命令行入口；
- [[LLMOps/03-项目与自测/examples/release_candidates.json|release_candidates.json]]：本地策略、基线和两个候选；
- [[LLMOps/03-项目与自测/examples/online_observations.json|online_observations.json]]：六种模拟 Canary 观察；
- [[LLMOps/03-项目与自测/examples/test_release_gate.py|test_release_gate.py]]：27 个标准库回归测试。

JSON 中的质量、安全、延迟、费用和漂移门槛只是为了让脚本产生可验证结果的**本地练习数值**，不是行业阈值、供应商价格或安全保证。

## 环境与运行

脚本只依赖当前稳定 Python 3 的标准库。先学习 `venv + pip` 的隔离方式，但把虚拟环境放到 vault 外，避免产生缓存和大量未跟踪文件：

```powershell
cd "X:\path\to\your-vault"
$venv = Join-Path $env:TEMP "llmops-learning-venv"
python -m venv $venv
& "$venv\Scripts\Activate.ps1"
python -m pip --version

$example = ".\Knowledge\AI Agent Engineer\docs\LLMOps\03-项目与自测\examples"
python "$example\release_gate.py" candidate --release-id release-safe
python "$example\release_gate.py" observe --observation-id obs-healthy
python "$example\release_gate.py" audit --release-id release-safe --observation-id obs-healthy
```

本项目不需要 `pip install`，不读取 `.env`，也不需要 API key。已有可用 Python 3 时可以不创建虚拟环境直接运行。

## 发布清单包含什么

一个候选必须固定：

- 应用 commit、API contract 和 SDK 锁文件摘要；
- Prompt 版本与内容摘要、上下文策略；
- 检索快照与配置；
- 供应商、模型快照和生成参数；
- 工具 schema 与实现；
- 护栏、数据处理、路由、降级和价格口径；
- 评测集、数据集、rubric 与 grader；
- 合同测试、任务/高风险切片、安全样本量、延迟和成本证据；
- Trace、脱敏、Canary、回滚与人工审批记录。

脚本拒绝 `latest`、`current`、`production` 等常见浮动标识，但这只是教学检查。真实系统仍需根据供应商官方文档、合同和内容摘要确认某标识能否重建当时行为。

## 预期结果

候选发布：

- `release-safe`：严格契约和全部本地门均满足，输出 `PROMOTE`，进程退出码为 `0`；
- `release-regression`：工具合同、高风险切片、安全样本量、费用和人工审批失败，输出 `BLOCK`，退出码为 `1`。

线上观察：

| 观察 | 预期动作 | 原因 |
| --- | --- | --- |
| `obs-healthy` | `CONTINUE` | 证据齐全且满足本地策略 |
| `obs-low-sample` | `INVESTIGATE` | 不能把样本不足误当成功 |
| `obs-provider-drift` | `FALLBACK` | 供应商信号退化且已验证降级路径就绪 |
| `obs-quality-regression` | `ROLLBACK` | 任务、高风险切片和安全门失败 |
| `obs-no-fallback` | `HUMAN_REVIEW` | 供应商退化但没有已验证备选 |
| `obs-capacity-cost` | `PAUSE` | 延迟与费用退化，先暂停扩大 |

没有指定 ID 时，命令会检查全部样例；只要其中存在阻断或动作要求，整体退出码就是 `1`。输入文件不存在、JSON 无效或契约错误时退出码是 `2`。

## 测试与验证

```powershell
$example = ".\Knowledge\AI Agent Engineer\docs\LLMOps\03-项目与自测\examples"
$env:PYTHONDONTWRITEBYTECODE = "1"
Push-Location $example
try {
    python -c "from pathlib import Path; [compile(p.read_text(encoding='utf-8'), str(p), 'exec') for p in (Path('release_gate.py'), Path('test_release_gate.py'))]"
    python -m unittest -v .\test_release_gate.py
    python -O -m unittest -v .\test_release_gate.py
    python -W error -m unittest -v .\test_release_gate.py
} finally {
    Pop-Location
}
```

这里用内存中的 `compile()` 做语法检查，不生成 `.pyc`。`-O` 会移除生产代码中的普通 `assert`，所以测试专门验证优化模式仍保持门禁。项目的生产判定不依赖 `assert`；测试中的 `self.assert*` 是测试框架断言，不会被优化模式静默删除。

## 分步实践

1. 运行健康联合审计，记录 policy 版本与 evidence 指纹；
2. 运行 `release-regression`，逐条映射到工具、质量、安全、成本和治理风险；
3. 将安全违规率保持为零，但把安全样本量改为 `1`，解释为什么仍被阻断；
4. 将模型快照改成 `latest`，确认这是输入契约错误（退出码 `2`），而不是普通候选失败；
5. 将 `obs-provider-drift` 的 `fallback_ready` 改为 `false`，确认不再自动降级；
6. 新增一个质量正常但 `redaction_check_passed=false` 的观察，确认隐私失败优先于性能信号；
7. 给策略新增“生效时间、风险理由、批准记录摘要”时，先扩展严格契约和测试，再修改 fixture。

## 自测题

1. 为什么 Prompt、API contract、工具 schema 和 grader 都应进入 manifest？
2. 为什么整体任务成功率上升时，高风险切片仍可以阻断发布？
3. 为什么“观察到零个安全违规”还需要检查样本量与覆盖范围？
4. 为什么 Trace 完整性是发布条件，而不只是事故后的可选功能？
5. 供应商漂移时，为什么脚本只允许切到 `fallback_ready=true` 的路径？
6. 为什么重新计算历史费用时必须固定当时的价格策略版本？
7. 证据指纹能证明什么，不能证明什么？

## 掌握检查

- [ ] 能解释严格 JSON 契约与业务门禁是两个层次；
- [ ] 能区分 `0/1/2` 三类退出码，并在 CI 中正确处理；
- [ ] 能解释每类候选检查对应的生产风险；
- [ ] 能把 Canary 观察映射为继续、调查、暂停、降级、回滚或人工复核；
- [ ] 能新增 fixture 和测试，并在普通、`-O`、warnings-as-errors 三种模式通过；
- [ ] 能区分“本地练习门满足”与“真实系统已安全”。

## 项目边界与下一步

本项目验证的是证据结构和决策逻辑；它没有调用真实模型、测量真实 token/费用、执行真实 Canary，也没有证明供应商、数据区域、线上流量或业务因果效果。证据指纹只证明同一输入可得到同一摘要，不能证明输入真实、完整或已经获批。

下一步可将相同清单接入 CI，并用 [[运行监控/00-目录|运行监控]] 建立真实观察面；任何真实供应商接入都要重新核对当前 API、价格、保留能力和合同。
