# 任务类型与图像表示

## 本节目标

先回答两个问题：你要模型做哪类任务，以及交付物到底由什么组成。任务说错，后面的提示再漂亮也难以验收。

## 从像素开始理解

数字图像是规则网格。每个格子是一个**像素**（pixel），通常记录红、绿、蓝三个通道的数值；带透明背景时还会有 alpha 通道。`1024 × 1024` 描述像素网格，不代表打印尺寸，也不保证画面清晰。宽高比（aspect ratio）是 `宽 ÷ 高`，它决定横版、竖版或方图的构图空间。

模型通常不会直接在最终大图的所有像素上反复计算。很多系统先把图像压缩到**潜空间**（latent space）：一个保留主要视觉结构、尺寸更小的数字表示，生成后再由解码器还原为像素。潜空间像“设计草图”，不是能被人直接观看的图片。

## 六类常见任务

| 任务 | 输入 | 输出 | 关键验收 |
| --- | --- | --- | --- |
| 文生图 | 文本 | 新图 | 提示遵循、构图、质量 |
| 图生图 | 图像 + 文本 | 保留部分结构的新图 | 与输入相似度、修改幅度 |
| 局部重绘（inpainting） | 图像 + 遮罩 + 文本 | 只替换指定区域 | 边缘融合、未遮罩区保持 |
| 扩图（outpainting） | 图像 + 新画布范围 | 向外补全 | 透视、光照、纹理延续 |
| 变体 | 图像 | 同主题多个版本 | 多样性与身份保持 |
| 超分辨率/修复 | 低清或受损图 | 更清晰版本 | 细节提升且不虚构关键事实 |

遮罩（mask）是一张与输入对齐的黑白或透明度图，用来指出“允许改哪里”。不同 API 对黑白含义、尺寸和格式的约定可能相反，接入前必须读当前文档并做小样测试。

## 需求澄清模板

不要只收集“画一张未来城市”。至少询问：用途与受众、画布比例、主体与动作、必须出现/禁止出现、文字是否要准确、参考素材权利、可接受的编辑范围、交付格式、尝试次数与人工审批人。每份参考素材还应有 `asset_id`、`source_revision`、允许用途和对象级授权/ACL；先授权再让评审器、模型或 Agent 读取。对新闻、医疗、证件或科学证据等用途，还要问“是否允许合成”，不能把生成图当事实记录。

## 常见错误与排查

- **把格式当质量**：PNG 不自动比 JPEG 清晰；它们是编码选择。
- **忽略下游裁切**：先确定投放比例，再构图，否则关键主体会被平台裁掉。
- **用生成做无损恢复**：生成式修复会补出“看似合理”的细节，不能宣称还原了真实信息。
- **编辑范围不清**：记录遮罩版本与“不许改变”清单，抽查未编辑区域。
- **把输出当证据**：生成画面可以是示意或创作材料；只有独立证据能支持的外部事实才可标为 `evidence_supported`。

## 练习与自测

1. “把商品图背景换成海滩，商品本体不变”属于哪类任务？需要哪些输入？
2. 竖版 `1080 × 1350` 的宽高比是多少？为什么不能直接写成 `4:5` 后就忽略像素尺寸？
