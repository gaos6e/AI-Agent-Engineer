# 公平、隐私、部署与排查

## 本节目标

识别语音数据的敏感性，按切片验证质量差异，并建立可回滚、可排障的服务边界。

## 为什么语音特别敏感

语音既包含说了什么，也可能暴露声纹、健康线索、环境和旁人的内容。获取同意、用途限制、最短保留期、访问控制和删除流程要覆盖原始音频、切片、转录、嵌入、日志和人工复核副本。把音频转成文本并不会自动消除隐私风险。

默认日志只记录请求 ID、时长、格式、阶段、延迟、错误码、版本和聚合指标，不记录原始音频、完整转录或可长期访问的链接。需要调试内容时采用有权限、短时、可审计的例外流程。

## 公平评测

总体 WER 可能掩盖某些语言、口音、性别、年龄、障碍或设备条件下的差异。只有在有合法目的、授权和足够保护时才收集敏感属性。报告每个切片的样本量、采样方式、置信区间和混杂因素，不把统计差异直接解释成生物或群体因果。

改进可以从数据覆盖、采集设备、VAD、分段、词表和人工替代通道入手。不能通过删除困难切片来“提高公平分数”。

## 部署与排障

锁定模型、解码、VAD、规范化和硬件配置；升级用固定回归集和小流量验证。常见现象：

| 现象 | 首查 |
| --- | --- |
| 静音仍输出文字 | VAD、无语音阈值、解码停止条件 |
| 长音频重复 | 窗口重叠、合并去重、重试幂等 |
| 句首常缺失 | VAD 起始缓冲、网络丢包 |
| 某设备突然变差 | 采样格式、增益、声道、固件变化 |
| 线上 WER 无法算 | 建立有许可的抽样标注与代理指标，不能用置信度冒充 WER |

## 练习与自测

- 写一个“录音何时删除、转录何时删除”的保留矩阵。
- 设计一个只用合成数据验证日志不泄露正文的测试。
- 如果一个切片只有 5 条样本，应该如何报告？明确样本量和高不确定性，不作强结论。
