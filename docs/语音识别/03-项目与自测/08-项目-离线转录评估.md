# 项目：离线转录评估

## 项目目标

使用人工编写的合成转录夹具，离线验证时间戳、说话人字段、WER/CER 和切片差异。项目不含音频，也不推理模型，适合在任何 Python 3 环境复现指标口径。

资产：

- [[语音识别/03-项目与自测/examples/asr_fixture.json|asr_fixture.json]]
- [[语音识别/03-项目与自测/examples/evaluate_transcript.py|evaluate_transcript.py]]
- [[语音识别/03-项目与自测/examples/test_contract_and_cli.py|test_contract_and_cli.py]]

## 运行

```powershell
Set-Location 'X:\path\to\your-vault\Knowledge\AI Agent Engineer\docs\语音识别\03-项目与自测\examples'
python -B .\evaluate_transcript.py .\asr_fixture.json
python -B .\evaluate_transcript.py --self-test
python -B -m unittest discover -s . -p 'test_*.py'
python -B -O -m unittest discover -s . -p 'test_*.py'
python -B -W error -m unittest discover -s . -p 'test_*.py'
```

脚本以 NFKC、`casefold()` 和移除标点后的空白 token 计算教学用 WER，以去空白字符计算 CER。此规则写在报告中，换规则就必须改变评测版本。当前回归集共 **63 项**，普通、`-O` 和 warnings-as-errors 三种模式都应通过。

## 输入与退出码合同

输入必须是 UTF-8 严格 JSON；重复键、`NaN`/`Infinity`、未知字段和错误类型会被拒绝。顶层精确包含 `schema_version`、`session_id`、`normalization` 与 `segments`，每段精确包含 ID、起止时间、说话人、切片、参考文本和预测文本。`speaker` 可为 `null`，表示未提供匿名说话人标签。

- 退出码 `0`：结构合同有效，且无重复 ID 或时间轴错误。
- 退出码 `1`：结构合同有效，但存在重复 ID、负时间、无效区间或重叠。
- 退出码 `2`：文件、UTF-8、JSON 或结构合同错误。

## 报告解释

- `micro_wer`/`micro_cer`：汇总编辑数除参考长度。
- `by_slice`：按夹具中的合成场景标签汇总；标签不代表真实人群。
- `timestamp_errors`：结束不晚于开始、段落倒序或重叠等契约问题。
- `audit_errors`：重复 `segment_id` 等跨段一致性问题。
- `speaker_coverage`：有匿名说话人标签的片段比例，不评价身份是否正确。

## 扩展任务

1. 把第二段起始时间改到第一段内部，确认出现重叠错误。
2. 为中文片段添加同义数字写法，比较不同 normalization 的影响。
3. 添加设备/噪声切片；每个切片同时报告参考词数，避免小样本强结论。
4. 增加“允许重叠语音”的显式字段，而不是一律报错。

## 验收标准

- [ ] 默认夹具退出码为 `0`，制造时间轴/重复 ID 错误时为 `1`，破坏结构合同时为 `2`。
- [ ] 63 项测试在普通、`-O` 和 warnings-as-errors 三种模式均通过，且不生成缓存。
- [ ] 能手算其中一个片段的 WER 并与脚本核对。
- [ ] 能解释脚本未验证真实音频质量、VAD、时间戳精度或说话人正确性。
- [ ] 能说明为何不应把合成 `slice` 结果当作真实公平结论。
