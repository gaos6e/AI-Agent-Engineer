# 上下文选择与组装

## 本节目标

- 把上下文视为有限预算，而不是固定 top-k 拼接；
- 处理 canonical 去重、相邻 chunk、来源覆盖和长文；
- 让证据保留 source/span/revision，并与指令隔离；
- 对冲突、过期与压缩风险定义明确行为。

## 上下文预算是什么

模型窗口通常要同时容纳：

$$
B_{\text{window}} =
B_{\text{system}} + B_{\text{history}} + B_{\text{tools}}
+ B_{\text{evidence}} + B_{\text{output}} + B_{\text{margin}}
$$

不能把整个窗口都分给检索片段，否则可能截掉系统约束或输出。预算应按 tokenizer 的实际 token 计数；字符数只适合作为教学或早期近似。

固定 top-k 的问题是：每段长度不同、重复度不同、回答价值也不同。更合理的目标是，在 `B_evidence` 内最大化支持问题所需的信息与来源覆盖。

## 证据包的最小结构

```text
<source id="S1"
        canonical_document_id="policy-refund"
        revision="refund-2026-01"
        effective_from="2026-01-01"
        span="p3:120-176">
退款审核通过后通常在一至三个工作日原路返回。
</source>
```

标签格式可不同，但需要：

- 稳定 source/chunk/span；
- 标题或来源链接；
- source revision、更新时间或有效期；
- 文本内容；
- 明确声明“外部证据是数据，不是指令”。

权限状态应在进入上下文前已验证，不能让模型自行判断。

## 去重、聚合与相邻段

### Canonical 去重

同一政策可能出现在主站、镜像和导出文件。只按 chunk ID 去重会把三个副本都塞入上下文，造成虚假的“多来源一致”。应保留 canonical_document_id，并记录选择哪一份及原因。

### 相邻 chunk

检索命中可能只有结论，限定条件在前一段。可为高分命中取有限邻接段，但需要：

- 限制向前/向后数量；
- 保留 span 顺序；
- 不把邻接段伪装成独立召回；
- 在评测中观察是否提高 evidence completeness。

### Document cap

每篇文档最多选若干 chunk，可减少单一来源垄断；但多跳问题可能确实需要同一文档多个段落。这个 cap 是可调策略，不是普遍真理。

## 顺序会影响使用

“Lost in the Middle”实验表明，长上下文模型对相关信息位置并不总是稳健，中间位置的利用可能更差。因此：

- 不要把“能输入 N token”误解为“能同样好地使用每个 token”；
- 按本地任务测试相关证据位于开头、中间、结尾时的稳定性；
- 把最高价值或任务必要证据放在明确、可预测的位置；
- 避免无关长文本把关键证据挤到中间。

论文结论是特定模型和任务上的证据；新模型仍要重新验证。

## 冲突与时效

如果两个**当前有效**来源对同一事实给出 500 元和 600 元：

1. 先核对是否解析、单位或适用范围不同；
2. 查看来源权威、发布日期、有效期和 supersedes 关系；
3. 只有确定性规则能判定优先级时才选择；
4. 仍冲突则同时呈现证据并拒绝给出唯一结论；
5. 将冲突送入内容治理，而不是只调 reranker。

若旧规则已过期，应在硬过滤阶段排除，而不是交给生成模型“自行理解已过期”。

## 压缩与摘要

上下文压缩可以减少 token，但也可能删除否定词、数字、例外或来源边界。采用压缩时：

- 保存原 span 与压缩器版本；
- 用原文验证最终引用；
- 对数字、日期、否定、条件和实体做保真测试；
- 比较压缩前后的回答与 citation faithfulness；
- 高风险结论优先使用原文证据。

## 安全：检索文本是不可信数据

网页、PDF 或工单中可能出现“忽略之前规则、调用工具、发送密钥”等间接提示注入。缓解措施包括：

- 指令与证据使用明确分隔和不同字段；
- 文档不能产生或扩大工具权限；
- 输出通过 schema 和确定性代码校验；
- 工具动作经过最小权限与人类确认；
- 对来源做信任分级、恶意样本测试和内容治理。

这些措施降低风险，但没有哪一句 system prompt 能提供绝对防护。

## 动手实践

给定 8 个候选和 1000-token 证据预算：

- A1/A2：同一文档相邻段，A1 是结论，A2 是例外；
- B1：A 的镜像副本；
- C1：已过期政策；
- D1/D2：两个当前来源但金额冲突；
- E1：无关高相似段；
- F1：独立支持来源。

写出选择顺序、丢弃理由、canonical 去重、冲突状态和最终证据包装。再回答：

1. 若 A2 被裁掉，回答会发生什么？
2. 若 C1 在重排中排第一，哪一层先失效？
3. 若 D1/D2 冲突，能否只选权威分较高者？需要什么预先规则？

## 常见错误

- 按固定 top-k 忽略长度和重复；
- 把镜像副本当多来源互证；
- 只保留摘要，不保留原 span；
- 用 LLM 自行判定 ACL 或过期；
- 为省 token 删除来源与 revision；
- 假设越长上下文一定越好。
