# 2026-07-21 第十三阶段 A2A 协议课程与安全评测生产化深审记录

## 阶段结论

本阶段新增一门以 A2A Protocol `1.0.0` 为合同基线的原创可选课程，并将其接入学习路线元数据、两条 Agent 角色路径、公开导航和构建验收；同时对安全/治理/隐私、评测/Benchmark/监控、MLOps/LLMOps/数据合成三组高风险课程完成多 Agent 深审。修改重点不是增加篇数，而是修复会让教学示例错误放行的合同缺口，并把动态规范、真实运行证据与离线教学验证明确分层。

本阶段也逐类审计了公开构建中的 346 个 fail-closed stub。它们全部来自受控第三方参考层，不是 346 篇空白原创课程，因此没有为了降低 stub 数量而复制正文、放宽发布策略或机械补写元数据。

> [!important] 结论边界
> 本阶段证明新增课程、所改示例、路线消费者和 Quartz 发布门在当前 fixture 下自洽；不证明真实 A2A 跨供应商互操作、线上身份与端点、密码协议、法规适用性、生产流量或所有历史页面已经完成深审。长期 `/goal` 继续保持 active。

## A2A 1.0 原创课程

新增 [[A2A/00-目录|A2A]] 课程，共 1 个入口、6 个专题章节和 1 个离线项目：

1. A2A 与普通 API、框架内编排、Tool Calling、MCP 的职责边界；
2. Agent Card 的发现、版本、能力声明、签名与信任建立；
3. Message、Task、TaskStatus、Part、Artifact 与终态/中断态；
4. JSON-RPC、gRPC、HTTP+JSON，流式、webhook、轮询与版本协商；
5. 认证、逐对象授权、凭据带外获取、多租户隔离和不可信内容；
6. 跨实现测试、`0.3`→`1.0` 迁移和 `observe/trial/adopt/hold` 采用决策；
7. 标准库离线合同验证器、4 组 fixture 与负向回归测试。

课程明确采用 A2A `1.0.0`，不把 `0.3` 的顶层 URL、`kind` Part 或旧枚举写成当前结构。验证器还拒绝重复 JSON 字段、非标准数值、无效 Base64、非 HTTPS 生产接口、终态回退和完成但无 Artifact；这些是规范子集与本地生产策略的教学验证，不是官方 SDK、Inspector、TCK、TLS/JWS/OAuth 或跨供应商一致性证明。

### 前沿域进入标准

`frontier-reference` 不再是新闻收集桶。主题必须同时具有一手规范或正式路线、可测试工程工件、与稳定主线的清晰边界、动态状态与退出条件，才可进入课程目录。A2A 1.0 满足这些条件，但只在独立 Agent 应用确实需要跨框架或跨组织互操作时加入，因此：

- `agent_app`：第 32 门，可选；
- `agent_platform`：第 38 门，可选；
- 硬前置为空，不强迫普通单 Agent、工作流或 MCP 项目采用；
- 总路线变为 57 门课程、10 个已填充知识域。

## 安全、治理与隐私深审

专项 Agent 深审并更新 25 篇 Markdown，重点统一以下边界：

- AI 安全：以真实资产、身份、对象、数据流、依赖和副作用约束补强 OWASP Agentic Top 10 映射；PDP/PEP 与对象级授权不能被粗粒度 scope、自报 `pinned` 或审批字符串替代；
- AI 治理：NIST AI RMF 的 Govern/Map/Measure/Manage 是迭代骨架而非一次性清单；补入 ISO/IEC 42005:2025 影响评估与 42006:2025 认证机构边界；
- 法规时效：欧盟 AI Omnibus 已完成议会一读与理事会最终通过，不能继续写成“仅政治协议”；正式公布、生效、合并文本和具体适用性仍须重新核对；
- 隐私计算：明确去标识、DP、联邦学习、安全聚合、MPC、同态加密与 TEE 各自保护对象和组合缺口；NIST Privacy Framework 1.1 仍是 IPD/coming soon，不冒充最终版。

三套离线项目均改为失败关闭：

- AI 安全 v2 合同绑定执行身份、具体 endpoint/resource、外发资产、memory flow、不可变依赖摘要和完整风险处置策略；
- AI 治理包把声明、证据、审批、变更和有效期绑定，拒绝以自报状态替代审计证据；
- 隐私审查器补强数据流/PET 适用性与跨字段一致性，避免局部字段“看起来安全”却与整体方案矛盾。

这些脚本仍依赖输入声明，没有连接真实 IAM、网关、制品库、数据血缘、密码实现或合规审计，因此输出 `PASS` 不能作为认证、法律或密码学保证。

## 评测、Benchmark 与运行可靠性深审

[[评测体系/00-目录|评测体系]]、[[Benchmark设计/00-目录|Benchmark 设计]] 与 [[运行监控/00-目录|运行监控]] 的主要修订包括：

- 把 judge 位置/长度偏差、提示注入、人工仲裁和版本冻结纳入评测设计；
- 明确 estimand、task/trial 层级、多重比较、等效性边界，以及 `pass@k`/`pass^k` 的独立同分布假设；
- 将污染拆成训练/后训练污染、运行时可发现性、开发者适配和维护链泄漏，区分 `not-detected` 与 `unknown`；
- 报告调整前后结果、提交预算、探索性分析和每成功任务成本，不用清理后的单一分数覆盖原始证据；
- 更新 OpenTelemetry GenAI 语义约定已迁移到独立仓库且仍为 Development 的动态状态；
- 补入“恢复不等于关闭”的事故闭环；修复监控 SLO `good_statuses` 未与事件枚举绑定的 fail-open，并拒绝重复 JSON 字段。

## MLOps、LLMOps 与数据合成深审

MLOps 离线晋级项目升级为 schema v2：

- 线上观察必须绑定已通过候选门的证据指纹、固定模型 ID 与制品摘要；
- 明确区分 shadow 与 Canary，shadow 故障阻断 rollout，Canary 故障才执行回滚；
- 同时校验总样本、有标签样本、关键切片样本和标签覆盖率，证据不足时只调查而不自动扩大；
- 决策指纹纳入 deployment/release，防止把线上判断移植到另一制品或旧策略；
- JSON 读取拒绝重复 key 与 `NaN` 等非标准常量。

LLMOps 持续评测章节删除已与当前代码脱节的 `fallback_ready` 练习，改为真实存在且带摘要绑定的 `fallback_evidence`。数据合成课程逐章复核后，确认隐私、偏差、训练/评测污染、真实留出、去重和效用边界已有清晰闭环，本阶段没有为了增加修改数量而重写。

MLOps 教学项目仍使用固定参考值，不是同期随机 control，也没有启动 Registry、真实 Canary、流量分配或因果效果验证；更严格的并行 control 证据继续由 LLMOps 项目承担。
