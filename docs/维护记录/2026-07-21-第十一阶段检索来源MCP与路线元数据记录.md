# 2026-07-21 第十一阶段检索、来源、MCP 与路线元数据记录

## 目标与证据边界

本阶段继续按“事实与安全错误 → 可运行实践 → 路线结构 → 表达体验”的顺序处理，没有按页面数量批量改写：

1. 逐页判断 RAG 组件、AI 安全与 AI 治理究竟是课程级硬前置，还是角色路线中的推荐顺序；
2. 对冻结的 LangChain Semantic Search 译文做来源、许可、39 个 Python fence 与当前 API 的独立终审；
3. 用原创、锁版本、可离线测试的 Retrieval Layer B 替代“复制上游片段即可运行”的错误学习路径；
4. 对 MCP 18 篇历史参考页逐页追溯仓库路径与许可切换，纠正内容形态和高风险事实；
5. 保持第三方/冻结参考层失败关闭，并用测试确认元数据重标不能绕过发布门禁。

三名只读子 Agent 分别审计路线、Semantic Search 与 MCP；另一名只读子 Agent复核 Retrieval Layer B 设计。主 Agent 统一决定术语、来源边界、修改范围、示例实现和最终验证，子 Agent 未直接修改或合并文件。

“离线示例通过”只证明固定依赖、合成数据与当前 adapter 合同；不证明真实 Embedding、远程 vector store、LangSmith、MCP host/SDK 或提供商服务已经端到端运行。

## 2. Semantic Search 严格终审

### 2.1 最强可追溯来源不是单一文件

本地译文由官方主 MDX 与两个 tab snippets 展开而成。最接近 2026-05-07 获取日期的可复现复合快照是 [`langchain-ai/docs@1dd0bd5a`](https://github.com/langchain-ai/docs/tree/1dd0bd5a13bb6e21589e897988ceb202d783e745)：

- `knowledge-base.mdx` blob `02bc453a836ea4d86d09f5d57569f7863723c99d`；
- `embeddings-tabs-py.mdx` blob `04137e51176697f2ce760e70f786ad4e52c3a984`；
- `vectorstore-tabs-py.mdx` blob `4a2d460f4ee40953d92e9292c9dbb7afeeac4c4e`；
- MIT LICENSE blob `c38f6f284dc464af69e9f618bc0304d299d0bdf0`。

这些文件与本地结构及原始特征错误高度吻合，但抓取时没有保存 deployment revision、响应头或响应哈希，所以只能写“可追溯到复合 blobs”，不能声称线上部署精确对应该 commit。

### 2.2 运行与许可阻断

- 39/39 Python fence 可由 `ast.parse` 解析，但普通 `.py` 只编译 38/39；一个顶层 `await` 仅适合 notebook/支持 top-level await 的 REPL。
- 15 个 Embedding 与 11 个 vector-store tabs 被展平成一条线性文档，会反复覆盖 `embeddings` / `vector_store`；固定输出无法同时对应这些互斥分支。
- PDF 不在仓库，安装说明缺少 `langchain-text-splitters` 等依赖；OpenSearch、Astra DB、MongoDB、PGVector 与 Pinecone 等分支还有缺失导入、变量、数据库或凭据。
- `text-embedding-3-large` 默认维度、`DeterministicFakeEmbedding` 与固定 1536 维输出互相矛盾。测试桩不具备语义能力，且当前实现需要未被 `langchain-core` 自动安装的 NumPy。
- 当前 `InMemoryVectorStore` raw score 是余弦相似度且越大越相似；通用 `similarity_search_with_score` 的语义仍由后端决定。`similarity_score_threshold` 使用 relevance score，不是任意 raw score。
- LangChain 仓库 MIT 可以覆盖其文档代码与说明，但不能当然覆盖页面复制的大段 NIKE 10-K 输出；公开实质正文还必须保留 `Copyright (c) 2025 LangChain` 与 MIT 文本。

因此 [[LangChain/02-LangChain/01-Semantic Search|Semantic Search]] 继续为 `third-party / needs-review`，公开站只生成质量待复核 stub。本地页新增复合来源、恢复门禁和四类阻断说明；没有为了减少 stub 数量恢复全文。

## 3. 原创 Retrieval 主线与 Layer B

[[LangChain/00-初学者路线/04-Retrieval与RAG组件|Retrieval 与 RAG 组件]]完成以下重写：

- 用 Mermaid 分开离线摄取与在线查询，并把受信任身份、ACL/filter 下推放在候选离开数据边界之前；Reranker 只接触已授权候选；
- 区分 raw similarity/distance、`[0,1]` relevance score、threshold retriever 与只返回 `Document` 的 Retriever；
- 记录 `InMemoryVectorStore==1.4.9` 没有 relevance 映射，调用该路径会 `NotImplementedError`，不能因基类有方法就推断支持；
- 补充远程 Embedding 的正文外发、远程 vector store metadata 外发、恶意文档、LangSmith trace、服务端 filter 下推和本地持久化副作用；
- 说明多数 top-k 对无关问题仍会返回候选，“无答案”需要经标注集校准的后端特定门或独立充分性判断。

新增 `docs/LangChain/00-初学者路线/examples/retrieval_layer_b/`：

| 文件 | 作用 |
| --- | --- |
| `in_memory_retrieval.py` | 透明三维教学 Embedding + 真实 `InMemoryVectorStore`；稳定 ID、完整 metadata、filter、raw score、Retriever 与 JSON CLI |
| `test_in_memory_retrieval.py` | 17 项正/负向测试，含重复/缺失 ID、metadata、跨 tenant、score、`invoke/batch`、CLI 与 AST 无运行时 `assert` |
| `requirements.txt` | 固定 `langchain-core==1.4.9` 与 Python 3.11 兼容的 `numpy==2.4.6` |

示例在添加前拒绝空/重复 ID、缺 metadata、`chunk_id != Document.id` 和非法 scope；添加后用 `get_by_ids` 按 ID 建表验证完整往返，不依赖返回顺序。查询 filter 是 InMemory 的进程内 callable，明确不冒充远程数据库 ACL。调用范围使用 `tracing_context(enabled=False)`，避免全机 tracing 配置让离线教学意外外发 trace。

17 项测试已在 Python 3.11.9 的隔离依赖环境中通过 normal、`-O`、warnings-as-errors 与组合模式；CLI 输出 `langchain-core 1.4.9`、NumPy 2.4.6、唯一授权命中 `alpha:refund:v1`，以及两条 query 对应的稳定 batch ID。精确浮点只检查有限性、方向和数学上应为 1 的教学向量，不保存脆弱的长浮点快照。

## 4. MCP 18 页来源、许可与事实审计

### 4.2 已修 P1

- Elicitation 明确分 form 与 URL mode；form 不得请求密码、API key、access token 或支付凭据。
- Roots 继续明确为协调范围，不是 sandbox/ACL；真实文件边界由 host、OS 权限和沙箱强制。
- 本地 server 的用户确认 UI 属于具体 host，不是 MCP 自动保证的权限模型。
- 远程页面把 Claude Custom Connectors 标为厂商语境；Streamable HTTP session 是 server 可选分配，不是所有远程连接必备。
- 固定来源中的组合 skills 是 `build-mcp-server`、`build-mcp-app`、`build-mcpb`，不再虚构 client/debug/deploy skills。
- Session hijacking 与 prompt injection 被拆成两条威胁链；前者使用 session ID、用户绑定、TLS、Origin、过期、撤销和重认证控制。
- Example Clients 上游页已于 2026-05-27 删除；原 URL 虽最终返回 200，却重定向到入门页。页面改用固定 blob，并明确只是一份历史生态快照。

发布 stub 对 `curated / needs-review` 参考页现在使用质量/来源待复核原因，而不是误报为“完全缺元数据”；该变化只改善说明，不放宽门禁。
