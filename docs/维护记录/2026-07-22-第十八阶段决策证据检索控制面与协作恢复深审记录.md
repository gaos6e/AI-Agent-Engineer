主 Agent 先固定“请求/返回模型身份、证据摘要、ACL-before-score、状态/lease 版本、人工对账”这些跨课术语，随后安排不重叠的专项：

1. 安全、治理与隐私：动态 MCP/Tool/Agent 组件、发布证据、RAG 删除和授权传播；
2. 工作流、多 Agent 与环境型 Agent：消息幂等、冲突冻结、恢复/lease/fencing 和浏览器可行动性；
3. Chunking、向量、Embedding、向量数据库、语义搜索和重排：输入空间、候选窗口、指标、删除/更新与多实例陈旧读；

主 Agent 随后审阅每项共享工作区改动、检查日期与术语一致性、修复模型选择目录断链，并在所有专项结束后重跑相关代码的普通与 `-O` 回归。没有 Agent stage、commit、push 或回退用户已有工作区状态。

## 检索基础：从数学演示升级为身份、版本和完整窗口合同

检索课程不再只以 top-k 结果说明安全或质量：

- Chunking 明确递归字符切分的默认边界更贴近空格分词语言；CJK/泰文等语料必须补充适合的分隔符并用 fixture 验收，token 长度函数不能替代边界设计。
- Embedding 明确不同模型的 query/document role、任务指令和自动截断语义不能互相照搬；从旧向量空间迁移时要全量重编码、双空间隔离和重新验证阈值。教学评测器拒绝重复 ranked ID 与非法 relevance grade，避免伪造大于 1 的 nDCG。
- 向量数据库项目将“upsert 会解决并发”替换为显式 `expected_source_revision` CAS、同 revision 异内容冲突、delete-before-create fence、resurrection token、tombstone 和 stale-read fail-closed。它同时说明当前 tombstone 不是永久审计账本，source identity 重用会造成 ABA；JSON 文件在 check 与替换之间仍有 TOCTOU，不能声称 snapshot isolation。
- 语义搜索的安全审计扩展到每条召回通道的完整 candidate/rank window，而不是仅展示的 top-k；未授权候选即使最终被截断，也可能已污染融合、缓存或 trace。
- Reranking 课程加入 candidate recall 的上界边界：某些评测器会补入正例，适合诊断排序上限而不构成端到端发布结果；发布时必须冻结真实 first-stage candidate IDs/window。

这些课程在目录、项目和正文中统一了 `content_origin` / `content_status`、复核日期、测试数和“教学实现不等于生产系统”的边界。

## 安全、治理、协作与恢复：让冲突停下来、让证据找得到

安全课程现在要求把动态发现的 MCP/Tool/Agent 组件保存为可复核快照：身份/发布渠道、协议/传输、schema 摘要、执行身份/scope、Roots/数据范围与出站目的地均进入变更触发器，但 bearer token 等秘密绝不进入快照。快照、回归结果和批准条件通过 `release_id` 和完整门禁摘要交给发布，摘要只用于变更检测，不是签名、授权或来源真实性证明。

治理课程补了离线评测、发布门、运行证据和人工分诊回归的最小交接表；隐私课程补了 RAG/Agent 的 chunk、Embedding、索引 generation、缓存、引用和记忆引用的 ACL、撤权、删除传播与 Tool Calling 再授权边界。两者都拒绝把“来源 hash”“来自内部知识库”或“删了一个向量”当作当前对象级授权或删除完成的证明。

多 Agent 离线模拟器修复了真实可复现缺陷：旧实现会把 `(task_id, idempotency_key)` 的不同 payload 静默视为重复，且允许 pending 任务直接验收外部结果。现在结果只能在 `running` 状态接受，并用 canonical JSON digest 区分安全重放与冲突；同 key、不同 digest 会记录两份摘要、升级为 `needs_review` 并阻断尚未开始的下游任务。该课程清楚标注 `needs_review` 是本库内部协调状态，不冒充 A2A 标准状态。

工作流和环境型 Agent 统一采用 `owner_worker`、`lease_version`、`expires_at`、`state_version`、CAS 与外部 fencing 的术语，区分运行所有权、状态提交和外部副作用。浏览器课程补充 Playwright 动态 locator / `locator.all()` 的等待边界，并强调 DOM 新鲜度不等于授权仍有效。
