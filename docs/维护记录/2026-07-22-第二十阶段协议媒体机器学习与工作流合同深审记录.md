# 2026-07-22 第二十阶段协议、媒体、机器学习与工作流合同深审记录

## 阶段结论

本阶段把七门相邻课程中最容易被误读为“格式正确、协议连通或模型生成成功就可以执行/发布”的边界收敛为可验证的合同。HTTP/JSON、Agent 委派、传统机器学习、生成媒体和确定性工作流现在都明确区分：输入可解析、结构/业务合同有效、可信主体已授权、副作用有回执，以及最终 outcome 已被核对。学习路径因此不再把 SDK、协议 capability、模型输出、哈希或内容凭证误当成业务成功、事实证明或权限。

## 多 Agent 分工与主 Agent 复核

主 Agent 先固定“结构化输入不等于可执行动作”“协议 ID 不等于可信身份”“回执不等于 outcome”“媒体来源不等于事实/授权”的跨课标准，再划分不重叠范围：

1. 主 Agent：[[API/00-目录|API]] 与 [[JSON/00-目录|JSON]] 的协议、重试、结构化输出和课程入口；
2. Agentic 模式：只审原创新手路线与离线项目，冻结且许可不明的参考层不改写；
3. 机器学习：主线、离线分类项目及深度学习衔接参考；
4. 图像/视频：媒体资产治理、发布证据和两个离线项目；
5. 工作流自动化：触发、契约、调度、审批、观测和离线 DAG 项目。

主 Agent 逐项复核 Agentic 的 MCP/A2A 规范表述、媒体动态公告、机器学习跨版本行为、工作流事件身份和所有项目测试数；各 Agent 没有 stage、commit、push 或改写其他课程。工作流目录中开始时已有的 `05-持久状态、恢复与幂等.md` 用户改动被明确隔离，未由本阶段改写或纳入专项结论。

## 关键改动

### API 与 JSON：把传输、表示和执行的门分开

- Requests 的 `SSLError` 虽是 `ConnectionError` 子类，但证书链/主机名身份失败不再进入重试分支；可靠客户端新增回归，课程用决策图说明 TLS、未知副作用、幂等查询和 `Retry-After` 的不同处置。
- API 示例现在按解析后的 media type 接受 `application/json` 与 `+json`，并在分页教学中检查成功状态、JSON object、元素形状、cursor、`max_pages` 与 `max_items`，避免把“不像 JSON 的成功响应”或单页膨胀交给下游。
- JSON 课程补出“完整且有界解码 → 严格解析 → 领域 schema/profile → 事实/版本/业务不变量 → 授权/allowlist → 审批/幂等 → outcome”的 Mermaid 门；模型、MCP 或外部来源给出的 JSON 只通过其中部分门。

### Agentic 模式：协议能力不会接管授权，批准必须晚于证据

- 新手路线增加 Tool Calling、MCP 与 A2A 的职责表：本地能力调用、标准化工具/资源连接、独立 Agent 委派可以组合，但都不替代模型外的身份、对象级授权、审批、幂等和 receipt/outcome 核验。
- 按当前 MCP Authorization 和 A2A 规范补充 resource/audience、禁止 token passthrough、逐请求/对象范围授权以及 `TASK_STATE_AUTH_REQUIRED` 只是请求额外授权而非已获人工批准。
- 可恢复工作流状态升至 v3：高风险路径先完成只读检查、再持久化等待审批；审批指纹绑定动作、检查证据、风险和策略版本，首次调用传入批准决定会失败关闭，重复 JSON key 和回执 key 均会拒绝。

### 机器学习：纠正实际版本回归，并把深度学习速查收敛为可验证的接口边界

- `ticket_router.py` 移除了在 scikit-learn 1.7.1 的 warnings-as-errors 下触发警告的显式 `l1_ratio=0.0`；课程锁定的直接依赖仍是 1.9.0，但明确本机只做了 1.7.1 兼容性验证，不能替代锁定环境重建。
- 数据切分、嵌套选择、校准/阈值、漂移、群体切片、公平性和隐私最小化的界限被补回主线。
- 重写损失函数和优化器参考，明确 logits/标签/shape、CrossEntropy 与 BCE-with-logits、loss 与业务指标、Adam/AdamW/weight decay 的框架差异；Transformer 页分清原论文 post-norm 与教学图 pre-norm，Mamba 页补齐上游 Linux/Python/PyTorch/CUDA 运行条件与复杂度不等于实测吞吐的边界。
- 站点元数据门禁还捕获并修复四页的非法 `content_origin` / `content_status` 枚举，现如实标为原创、已核验内容。

### 图像与视频：媒体资产有来源链，不因凭证或生成结果自动成为事实

- 两门课程和项目统一 `asset_id`、`source_revision`、`transform_id`、`release_id`、`acl_reference` 与撤权/删除传播计划；素材必须在预览、评分或模型调用前通过对象级 ACL。
- C2PA/Content Credentials 被限定为可验证的来源与完整性声明，不证明事实真实性、人物身份、权利或业务授权；`evidence_bound` 与 `evidence_supported` 继续分开。
- 视频目录把 Sora 2/API 的关闭准确表述为 2026-07-22 访问官方页时公告的**未来计划**（目标 2026-09-24），而非既成状态。
- 图像任务审计器与视频任务包验证器新增资产 revision、ACL、删除传播、事实证据、预算与发布链路的硬门禁及回归。

### 工作流自动化：事件身份可去重，但不是调用者身份或授权

- 离线引擎不再用 `source + "::" + id` 拼接 CloudEvents 身份，以规范 JSON 二元组编码避免分隔符碰撞；可选 `time` 收窄为标准库可可靠解析、带显式 UTC offset 的 RFC 3339 子集，并明确它不是通用 CloudEvents 网关。
- 入口教学顺序收敛为有界原始输入、签名/mTLS/认证、新鲜度/nonce、契约、结构化去重、持久创建、异步执行；每次执行仍按可信 actor、策略和资源 ACL 授权。
- 课程统一 `event identity`、`instance_id`、`operation_id`、`attempt_id`、`trace_id`、receipt 与 outcome 的不同作用域，并强化条件汇合、未知结果、补偿、审批原子消费、版本化和观测边界。

## 跨课程术语与责任边界

| 术语 | 本阶段的稳定含义 | 不可错误替代 |
| --- | --- | --- |
| 可解析 JSON | 文本可在受限资源下解码并通过语法 | 领域 schema、事实核验、授权或执行许可 |
| protocol/capability ID | 一次协议请求、工具/资源能力或远端 task 的关联信息 | 可信 actor、tenant、对象权限或本地 action ID |
| `event identity` | 经认证后用于判定重复事件的结构化 `(source, id)` | webhook 签名、调用者身份、授权结果或去重保留策略 |
| `operation_id` / `attempt_id` | 前者代表稳定业务意图，后者代表一次领取、投递或重试 | 彼此、receipt、trace ID 或业务 outcome |
| receipt / outcome | 前者是外部动作证据，后者是已核对的最终业务状态 | “已接收”、HTTP 成功码、远端 `completed` 或模型自述 |
| `source_revision` / `transform_id` / `release_id` | 分别标识冻结输入、一次处理/变换和可审计发布候选 | 内容真实性、当前授权、数据集版本或模型别名 |
| Content Credentials | 对某个声明/历史的来源与完整性信号 | 人物同意、版权/访问授权、事实真实性或发布批准 |
| approval fingerprint | 绑定动作、证据、策略和状态的防偷换指纹 | 审批者认证、签名、一次性消费事务或下游授权 |
