# CrewAI 学习目录

## 知识库简介

CrewAI 是一个 Python Agent 编排框架，提供两组互补抽象：**Crews** 用 Agent、Task 和 Process 组织协作任务；**Flows** 用事件、路由和状态控制应用流程。官方 PyPI 描述明确区分二者：Crews 偏向自主协作，Flows 偏向细粒度、事件驱动的执行控制。

本库不把“多 Agent”当成默认答案。先确定任务、状态、工具权限和完成条件，再选择普通 Python、Flow、Crew 或两者组合。七节课程从安装和核心对象讲到状态、工具、记忆、评测、恢复与生产边界，最后完成一个不需要 CrewAI 包、模型或密钥的离线项目。

> [!important] 版本事实分成两层
> - **可安装包快照**：PyPI 在 2026-07-14 显示最新稳定版 `crewai 1.15.2`，发布于 2026-07-08，声明要求 Python `>=3.10,<3.14`。
> - **文档页面标签**：同日访问时，首页/Agents 显示 `v1.14.6`，Flows 显示 `v1.14.5`，Installation/Tools 显示 `v1.14.0`，Tasks 显示 `v1.12.1`，Memory 显示 `v1.12.2`。
>
> 页面标签不一致不足以证明某段示例适用于 `1.15.2`。真实项目必须锁定安装版本，再用该版本源码、API Reference、最小导入测试和集成测试复核。本文记录的是 2026-07-14 的学习基线，不是永久 API 保证。

## 在总路线中的位置

本库位于“框架实践”阶段。应先理解 [[Agent 核心/00-目录|Agent 核心]]、[[Tool Calling（含 Function Calling）/00-目录|Tool Calling（含 Function Calling）]] 和 [[Agentic Design Patterns/00-目录|Agentic Design Patterns]]，再把框架无关的循环、工具、状态、审批和评测映射到 CrewAI。

## 学习目标

- 能解释 Agent、Task、Crew、Process、Flow、State 和 Event 的职责。
- 能在普通 Python、Flow、Crew 和 Flow + Crew 之间选择最低复杂度方案。
- 能用 Task 契约与结构化输出隔离 Agent 之间的数据边界。
- 能区分 Flow state、当前 context、Memory 与 Knowledge。
- 能为工具设置最小权限、参数校验、错误分类、幂等和审批。
- 能设计离线单元测试、轨迹评测、真实模型试验和生产门槛。
- 能识别文档/包版本漂移，并验证而不是猜测 API。

## 前置知识

- Python 函数、类、装饰器、类型提示、异常、JSON 和基础 `unittest`。
- 知道 LLM 调用会产生费用且输出有波动；理解 Tool Calling 和基本 Agent 循环。
- 会在 Windows 11 PowerShell 7 中创建虚拟环境。

## 安装路线：先 venv + pip

不要在 Obsidian vault 内创建 `.venv`。在独立练习项目目录中，以兼容范围内的 Python 3.13 为例：

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install "crewai==1.15.2"
python -c "from importlib.metadata import version; print(version('crewai'))"
```

只有确实需要额外工具集时再安装：

```powershell
python -m pip install "crewai[tools]==1.15.2"
```

上述版本号是本次核对快照；以后新建项目应重新查 PyPI、发行说明和 Python 约束。官方 Installation 页面当前主推 `uv` 和 CrewAI CLI；先理解 `venv + pip` 后，可按官方文档改用 `uv` 管理项目，但不要在同一环境里无计划地混用两个锁定来源。

真实模型提供商通常需要凭据。本库不提供、不读取也不验证真实密钥；只在独立项目中使用环境变量或密钥管理服务，绝不把值写进 YAML、Markdown、测试夹具或 Git。

## 动手实践或项目入口

综合项目使用三个确定性 Task 桩模拟研究、写作与审核，再由外层 Flow 控制修订预算、终态和发布。文件位于：

- `examples/offline_research_flow.py`
- `examples/sources.json`
- `examples/test_offline_research_flow.py`

项目不导入 CrewAI、不联网、不调用模型；它验证的是架构契约。真实 CrewAI API 片段只作为当前官方示例的学习映射，本轮未在安装 `crewai==1.15.2` 和配置提供商的环境中执行，因此不声称真实端到端已验证。

## 掌握标准

- [ ] 能在一句话内区分 Crew 与 Flow，并给出各自不适合的任务。
- [ ] 能为 Agent 设置能力和工具边界，为 Task 写可验证产物。
- [ ] 能解释 sequential 与 hierarchical Process 的额外成本和要求。
- [ ] 能用结构化 state 和有限路由控制循环与终态。
- [ ] 能区分 Memory、Knowledge、state 与当轮 context。
- [ ] 能让写工具经过执行层校验、审批和幂等保护。
- [ ] 能在普通、`-O`、warnings-as-errors 三种模式运行离线测试。
- [ ] 能记录包版本、页面标签、模型、工具 schema、数据与评测集版本。

## 与其他知识库的关系

- [[Agentic Design Patterns/00-目录|Agentic Design Patterns]] 提供框架无关的路由、并行、审批、恢复和评测模式。
- [[LangChain/00-目录|LangChain]] 与 LangGraph 是另一组组件和状态编排抽象；比较时应使用同一任务集、模型和成本口径。
- [[MCP/00-目录|MCP]] 可为 Agent 暴露远程工具，但协议连接不等于工具已获授权或可信。
- CrewAI 只负责部分运行时能力；LLMOps、监控、安全和治理仍需独立工程设计。
