# AI Agent Engineer 网站设计规范

## 品牌定位

- 名称：AI Agent Engineer
- 副标题：从零构建、评测与部署 AI Agent
- 核心感受：严谨的工程内容放在轻快、开放的编辑式画布上；黑白负责秩序，粉彩色块负责叙事。
- 视觉来源：依据 `DESIGN-figma.md` 转译为中文技术知识库，不复制 Figma 品牌资产或专有字体。

## 信息架构

- 首页：白色 Hero、黑色数据带、lime 阶段系统区、白色学习重点区、navy 公开边界区；阶段与课程数量从当前课程索引动态计算。
- 完整路线：保留唯一的 Dataview 主课程地图，并允许地图标题和使用说明、角色路径、内容分层、进阶方向、维护说明随信息架构演化。
- 文档页：顶部品牌与搜索、左侧阶段树、正文、右侧页内目录与反向链接。
- 左侧树：阶段 → 学习重点 → 真实子目录 → 页面；阶段、课程和目录均使用原生 `details/summary`。
- 移动端：课程树进入带遮罩的抽屉，支持 Escape、焦点回归和背景滚动锁定。
- 不提供登录、评论、云端同步或学习进度。

## 设计令牌

### 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--aae-canvas` | `#FFFFFF` | 页面画布、白色卡片 |
| `--aae-ink` | `#000000` | 标题、主按钮、当前状态 |
| `--aae-hairline` | `#E6E6E6` | 分隔线、表格与控件边界 |
| `--aae-surface-soft` | `#F7F7F5` | 列表悬停和弱表面 |
| `--aae-lime` | `#DCEEB1` | 学习系统主色块 |
| `--aae-lilac` | `#C5B0F4` | 当前课程、编辑式便签 |
| `--aae-cream` | `#F4ECD6` | 暖色内容区 |
| `--aae-coral` | `#F3C9B6` | 评测/交付便签 |
| `--aae-navy` | `#1F1D3D` | 反色叙事区 |

规则：默认画布不使用渐变；大色块不加阴影；一屏尽量只出现一个主色块；主操作使用黑色胶囊，粉彩色只承担叙事和状态。

### 字体

- Display/Sans：`Inter` → `Segoe UI Variable` → `SF Pro` → `Noto Sans SC` → `Microsoft YaHei UI` → `system-ui`。
- Mono：`JetBrains Mono` → `Cascadia Code` → `SFMono-Regular` → `Consolas`。
- 专有 `figmaSans`/`figmaMono` 不进入项目。
- 正文保持约 17px、行高 1.8；界面文字、导航、元信息和标题整体高于旧版本。
- 首页主标题最高约 86px；文章标题、H2、H3 依次形成明显编辑式层级。
- Mono 只用于阶段编号、标签和统计，不用于正文段落。

### 形状与间距

- 基础节奏：8px。
- 大段落间距：72–96px。
- 主色块圆角：24px；移动端全宽色块取消边缘圆角。
- 普通容器圆角：8px；文字按钮统一胶囊；图标按钮统一圆形。
- 触控目标不少于 44×44px。

## 组件规则

- 顶部导航：平面 canvas、hairline 下边线、64px 视觉高度；不使用玻璃卡片阴影。
- Hero：左侧大标题与操作，右侧四张轻微旋转的粉彩便签；便签只作视觉叙事，不承载关键操作。
- 数据带：纯黑底白字，展示阶段、课程、源文档和公开正文。
- 阶段系统：lime 大色块中放置白色阶段卡；卡片 hover 只做轻微位移和黑色边界。
- 学习重点列表：白色画布上的扁平行项目，不为每个课程制造高阴影卡片。
- 左侧树：阶段字号最大，课程次之，目录与页面再次递减；只有一个滚动容器，滚动条外观隐藏但滚轮、触控与键盘滚动保留。
- 当前页面：黑底白字；当前课程：lilac 状态面。
- 搜索和资源预览：唯一允许使用明显浮层阴影的区域。

## 响应式

- `>1200px`：三栏阅读，左栏 19–22rem、正文 36–50rem、右栏 15–18rem。
- `<=960px`：顶部隐藏普通导航链接，显示课程目录按钮；左栏树进入抽屉。
- `<=800px`：首页色块全宽，Hero/路线/公开边界改为单列。
- `<=560px`：CTA 全宽、阶段卡和课程入口单列、主标题约 48–64px。
- 所有断点禁止页面级横向溢出；表格和代码块可在自身容器内滚动。

## 动效

- GSAP 仅做渐进增强；无 JavaScript 时完整内容仍可使用。
- 首页使用 8–24px 的短距离 `y + autoAlpha` 入场，时长 220–580ms；不使用弹跳和大位移。
- 粉彩便签和内容区使用 stagger；只动画 transform 与 opacity。
- `ScrollTrigger.batch()` 只执行一次，不 pin、不劫持滚动。
- SPA `prenav` 清理全部动画，`nav` 后重新初始化。
- `prefers-reduced-motion: reduce` 时跳过 GSAP 动画，并移除非必要 CSS 过渡。

## 可访问性与状态

- 原生 `details/summary` 提供 Enter/Space 展开语义；链接使用 `aria-current="page"`。
- 移动抽屉使用 `aria-controls`、`aria-expanded`、`aria-hidden` 和 `inert`，并实现焦点圈定与恢复。
- 搜索、主题、关闭、资源预览都提供可访问名称。
- 焦点环使用当前 ink 的 3px 高对比轮廓，不只依赖颜色变化。
- 明暗主题都必须验证正文、粉彩色块、当前项和焦点对比。
- 发布校验必须拒绝含 `span.katex-error` 的生成页面，避免把无法阅读的原始公式作为成功构建发布。

## 明确不做

- 不引入在线字体、Figma 专有字体或品牌图形。
- 不使用渐变文字、粒子背景、WebGL、滚动劫持或无限动画。
- 不为视觉统一改写课程正文。
- 不把许可不明确的第三方完整复刻或 Vault 其他区域带入公开产物。
